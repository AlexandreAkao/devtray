# DevTray v0.8 — Release / Distribution infra (Sparkle + release CI, ad-hoc beta) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make DevTray installable and self-updating for other people via Sparkle auto-update + a tag-triggered GitHub Actions release pipeline, all under ad-hoc code signing (a "beta"; Developer ID + notarization is a future slice).

**Architecture:** Sparkle 2.x is added as an SPM dependency linked into the app target; a small `UpdaterController` (`ObservableObject` wrapping `SPUStandardUpdaterController`) drives a "Check for Updates…" action surfaced in the Settings About tab and the popover footer (the footer gets the action via an injected closure so DevTrayUI never imports Sparkle). A `release.yml` workflow fires on `v*` tags: build Release unsigned → ad-hoc sign with hardened runtime → zip → EdDSA-sign with Sparkle's `sign_update` → publish a GitHub Release → append a signed `<item>` to `docs/appcast.xml` (served by GitHub Pages) and commit it back to `main`. Separately, `test.yml`'s stale package matrix is replaced with a single-source dynamic matrix driven by a new `scripts/test.sh --list` mode.

**Tech Stack:** Swift 5.10, SwiftUI, Sparkle 2.x (SPM), XcodeGen, GitHub Actions (macos-14, Xcode 15.4), bash, `codesign`/`ditto`/`PlistBuddy`, `xmllint`.

**Spec:** `docs/superpowers/specs/2026-05-28-devtray-v0.8-release-infra-design.md`

**Branch:** `feat/v0.8-release` (already created in place — no worktree; manual smoke/packaging runs better on the main checkout). The design doc was committed here.

## Notes for the executor (read first)

- **The Xcode scheme is `DevTrayApp`, NOT `DevTray`.** Always build with `xcodebuild -project DevTray.xcodeproj -scheme DevTrayApp build`. `-scheme DevTray` fails with a misleading "unable to resolve module dependency: DevTrayCore" cascade.
- **Deployment target is macOS 14.0.** Sparkle 2.x supports macOS 10.13+, so 14.0 is fine.
- **Version lives in TWO files** — `project.yml` (`targets.DevTrayApp.info.properties`) **and** `App/DevTrayApp/Info.plist`. Edit both with identical values, then run `xcodegen generate`. After generate, `git diff App/DevTrayApp/Info.plist` to confirm the keys survived (XcodeGen regenerates the plist from `info.properties`; since both hold the same keys the result is consistent).
- **Run `xcodegen generate`** after every `project.yml` edit. The `.xcodeproj` is gitignored and generated.
- **Sparkle is an APP-LEVEL dependency**, so (unlike Yams, which is a leaf-package dep) its pin does NOT land in a tracked `Package.resolved` — it goes into the gitignored `DevTray.xcodeproj/.../swiftpm/Package.resolved`, exactly like the existing `KeyboardShortcuts`. To keep releases reproducible (Sparkle is the shipped auto-updater), Sparkle is pinned with `exactVersion:` in `project.yml` rather than `from:`. Nothing to commit beyond `project.yml`.
- **DevTrayUI must not import Sparkle.** The popover "Check for Updates…" item is wired via an injected closure + a `Bool`, not the Sparkle type.
- **Ad-hoc signing has no `DEVELOPMENT_TEAM`.** Local dev build is unchanged (ad-hoc Automatic). Release signing happens only in CI via `codesign --sign -`.
- **The placeholder public key:** until the AUTHOR runs `generate_keys` (Task 10), `SUPublicEDKey` is a `PLACEHOLDER` string. The app still builds and launches; the updater simply can't verify updates yet and "Check for Updates" stays disabled / errors on use. Full update smoke is gated on Task 10.
- **`gh` dual account:** before any `gh` API op from your machine (the PR in Task 11) run `gh auth switch --user AlexandreAkao`, restore after. SSH push works on either. The *release CI* uses the in-runner `GITHUB_TOKEN`, so the dual-account dance does not apply there.
- **Commit after each task.** Frequent commits; each task leaves the tree building.

import Foundation
import Combine
import Sparkle

@MainActor
final class UpdaterController: ObservableObject {
    private let controller: SPUStandardUpdaterController
    @Published var canCheckForUpdates = false

    init() {
        controller = SPUStandardUpdaterController(
            startingUpdater: true,
            updaterDelegate: nil,
            userDriverDelegate: nil
        )
        controller.updater.publisher(for: \.canCheckForUpdates)
            .assign(to: &$canCheckForUpdates)
    }

    func checkForUpdates() {

}

Create `scripts/appcast.sh`:
```bash
#!/usr/bin/env bash
# Append a signed Sparkle <item> to the appcast (newest-first), then validate.
#
# Usage: scripts/appcast.sh <shortVersion> <build> <url> <edSignature> <length> [appcastPath]
set -euo pipefail

SHORT="$1"
BUILD="$2"
URL="$3"
SIG="$4"
LEN="$5"
APPCAST="${6:-docs/appcast.xml}"
MIN_OS="14.0"
PUBDATE="$(LC_ALL=C date -u '+%a, %d %b %Y %H:%M:%S +0000')"

ITEM=$(cat <<EOF
    <item>
      <title>${SHORT}</title>
      <sparkle:version>${BUILD}</sparkle:version>
      <sparkle:shortVersionString>${SHORT}</sparkle:shortVersionString>
      <sparkle:minimumSystemVersion>${MIN_OS}</sparkle:minimumSystemVersion>
      <pubDate>${PUBDATE}</pubDate>
      <enclosure url="${URL}" sparkle:edSignature="${SIG}" length="${LEN}" type="application/octet-stream"/>
    </item>
EOF
)

# Insert the new item directly after the marker (BSD sed 'r' reads a file).
TMP_ITEM="$(mktemp)"
printf '%s\n' "$ITEM" > "$TMP_ITEM"
sed -e "/<!-- BEGIN ITEMS -->/r $TMP_ITEM" "$APPCAST" > "$APPCAST.tmp"
mv "$APPCAST.tmp" "$APPCAST"
rm -f "$TMP_ITEM"

xmllint --noout "$APPCAST"
echo "Added ${SHORT} (build ${BUILD}) to ${APPCAST}"
```

## Task 8: Release workflow (`.github/workflows/release.yml`)

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the workflow**

Create `.github/workflows/release.yml`:
```yaml
name: release
on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  release:
    name: build · ad-hoc sign · publish
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Select Xcode 15.4
        run: sudo xcode-select -s /Applications/Xcode_15.4.app

      - name: Install XcodeGen
        run: brew install xcodegen

      - name: Generate project
        run: xcodegen generate

      - name: Read version
        id: ver
        run: |
          PLIST="App/DevTrayApp/Info.plist"
          SHORT=$(/usr/libexec/PlistBuddy -c "Print :CFBundleShortVersionString" "$PLIST")
          BUILD=$(/usr/libexec/PlistBuddy -c "Print :CFBundleVersion" "$PLIST")
          echo "short=$SHORT" >> "$GITHUB_OUTPUT"
          echo "build=$BUILD" >> "$GITHUB_OUTPUT"
          echo "Version $SHORT (build $BUILD) for tag ${GITHUB_REF_NAME}"

      - name: Build (Release, unsigned)
        run: |
          xcodebuild \
            -project DevTray.xcodeproj \
            -scheme DevTrayApp \
            -configuration Release \
            -derivedDataPath build \
            CODE_SIGNING_ALLOWED=NO \
            build

      - name: Ad-hoc sign with hardened runtime
        run: |
          APP="build/Build/Products/Release/DevTray.app"
          codesign --force --deep --options runtime --sign - "$APP"
          codesign --verify --deep --strict "$APP"

      - name: Zip
        id: zip
        run: |
          APP="build/Build/Products/Release/DevTray.app"
          ZIP="DevTray-${{ steps.ver.outputs.short }}.zip"
          ditto -c -k --keepParent "$APP" "$ZIP"
          echo "zip=$ZIP" >> "$GITHUB_OUTPUT"
          echo "length=$(stat -f%z "$ZIP")" >> "$GITHUB_OUTPUT"

      - name: Download Sparkle tools
        run: |
          SPARKLE_VER="2.6.4"
          curl -fL -o sparkle.tar.xz \
            "https://github.com/sparkle-project/Sparkle/releases/download/${SPARKLE_VER}/Sparkle-${SPARKLE_VER}.tar.xz"
          mkdir -p sparkle-tools
          tar -xf sparkle.tar.xz -C sparkle-tools

      - name: EdDSA-sign the archive
        id: sign
        env:
          SPARKLE_PRIVATE_KEY: ${{ secrets.SPARKLE_PRIVATE_KEY }}
        run: |
          KEYFILE="$(mktemp)"
          printf '%s' "$SPARKLE_PRIVATE_KEY" > "$KEYFILE"
          OUT=$(./sparkle-tools/bin/sign_update -f "$KEYFILE" "${{ steps.zip.outputs.zip }}")
          rm -f "$KEYFILE"
          echo "sign_update: $OUT"
          SIG=$(echo "$OUT" | sed -n 's/.*sparkle:edSignature="\([^"]*\)".*/\1/p')
          if [ -z "$SIG" ]; then echo "::error::failed to parse edSignature"; exit 1; fi
          echo "sig=$SIG" >> "$GITHUB_OUTPUT"

      - name: Create GitHub Release
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          gh release create "${GITHUB_REF_NAME}" \
            "${{ steps.zip.outputs.zip }}" \
            --title "${{ steps.ver.outputs.short }}" \
            --generate-notes

      - name: Update appcast on main
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git fetch origin main
          git checkout main
          URL="https://github.com/AlexandreAkao/devtray/releases/download/${GITHUB_REF_NAME}/${{ steps.zip.outputs.zip }}"
          ./scripts/appcast.sh \
            "${{ steps.ver.outputs.short }}" \
            "${{ steps.ver.outputs.build }}" \
            "$URL" \
            "${{ steps.sign.outputs.sig }}" \
            "${{ steps.zip.outputs.length }}"
          git add docs/appcast.xml
          git commit -m "chore(release): appcast ${GITHUB_REF_NAME}" || { echo "no appcast change"; exit 0; }
          git push origin main
```

> **Note on the Sparkle tools version:** `SPARKLE_VER="2.6.4"` must match an existing release asset `Sparkle-X.Y.Z.tar.xz` at `github.com/sparkle-project/Sparkle/releases`. If the `curl` 404s, bump to the latest 2.x release tag. (`curl -fL` fails the step on 404 so this surfaces clearly.)
