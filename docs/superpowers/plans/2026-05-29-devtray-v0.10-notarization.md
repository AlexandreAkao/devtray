# DevTray v0.10 — Notarization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace ad-hoc code signing in `.github/workflows/release.yml` with Developer ID signing + `xcrun notarytool` + `xcrun stapler`, eliminating Gatekeeper first-launch friction for users downloading DevTray from GitHub Releases. Bump to `0.10.0` / build `10`.

**Architecture:** Pure delta on the v0.8 release pipeline. **No Swift code changes.** Replace two v0.8 steps (ad-hoc sign + zip) with 10 new steps (env setup → cert import → identity detect → Developer ID sign → verify → ASC key decode → notary zip → notarize → staple → re-zip), then append one cleanup step with `if: always()` to never leak secrets. Cert/key/team-name are not hardcoded — the signing identity is auto-detected as the SHA-1 hash from the imported keychain at runtime.

**Tech Stack:** GitHub Actions (macos-14 runner), Xcode 16.2 (`xcrun codesign`, `xcrun notarytool`, `xcrun stapler`), `security` CLI for keychain management, `jq` for JSON parsing (preinstalled on macos runners), Sparkle 2.9.2 EdDSA (unchanged from v0.8).

**Spec:** `docs/superpowers/specs/2026-05-29-devtray-v0.10-notarization-design.md`

**Branch (manual setup before Task 1):** `feat/v0.10-notarization` (in place; no worktree). Create with:
```bash
git checkout -b feat/v0.10-notarization
```

## Task 2: `release.yml` — replace ad-hoc + Zip with Developer ID sign + notarize + staple + re-zip

The atomic core of the cycle. Replaces lines 64–77 of `release.yml` (the v0.8 "Ad-hoc sign with hardened runtime" step + blank + "Zip" step) with 10 new steps in order: env setup, cert import, identity detect, Developer ID sign, verify, ASC API key decode, zip for notarytool, notarize, staple, re-zip (final shipping artifact).

- [ ] **Step 2: Replace lines 64–77 with the 10-step Developer ID + notarize block**

Current content (lines 64–77, the two steps + blank line between):
```yaml
      - name: Ad-hoc sign with hardened runtime
        run: |
          APP="build/Build/Products/Release/DevTrayApp.app"
          codesign --force --deep --options runtime --sign - "$APP"
          codesign --verify --deep --strict "$APP"

      - name: Zip
        id: zip
        run: |
          APP="build/Build/Products/Release/DevTrayApp.app"
          ZIP="DevTray-${{ steps.ver.outputs.short }}.zip"
          ditto -c -k --keepParent "$APP" "$ZIP"
          echo "zip=$ZIP" >> "$GITHUB_OUTPUT"
          echo "length=$(stat -f%z "$ZIP")" >> "$GITHUB_OUTPUT"
```

Replace with:
```yaml
      - name: Setup signing env
        run: |
          echo "KEYCHAIN_PATH=$RUNNER_TEMP/build.keychain-db" >> "$GITHUB_ENV"
          echo "APP_PATH=build/Build/Products/Release/DevTrayApp.app" >> "$GITHUB_ENV"

      - name: Import Developer ID certificate
        env:
          P12_BASE64: ${{ secrets.DEVELOPER_ID_P12_BASE64 }}
          P12_PASSWORD: ${{ secrets.DEVELOPER_ID_P12_PASSWORD }}
        run: |
          CERT_PATH="$RUNNER_TEMP/cert.p12"
          echo "$P12_BASE64" | base64 --decode > "$CERT_PATH"
          security create-keychain -p "" "$KEYCHAIN_PATH"
          security set-keychain-settings -lut 21600 "$KEYCHAIN_PATH"
          security unlock-keychain -p "" "$KEYCHAIN_PATH"
          security import "$CERT_PATH" -P "$P12_PASSWORD" -A -t cert -f pkcs12 -k "$KEYCHAIN_PATH"
          security set-key-partition-list -S apple-tool:,apple: -s -k "" "$KEYCHAIN_PATH"
          security list-keychain -d user -s "$KEYCHAIN_PATH" $(security list-keychain -d user | tr -d '" ')

      - name: Detect signing identity
        run: |
          IDENTITY=$(security find-identity -v -p codesigning "$KEYCHAIN_PATH" \
            | awk '/Developer ID Application/ {print $2; exit}')
          if [ -z "$IDENTITY" ]; then
            echo "::error::Developer ID Application identity not found in imported keychain"
            security find-identity -v -p codesigning "$KEYCHAIN_PATH"
            exit 1
          fi
          echo "IDENTITY=$IDENTITY" >> "$GITHUB_ENV"
          echo "Detected signing identity: $IDENTITY"

      - name: Sign app with Developer ID (hardened runtime + secure timestamp)
        run: |
          codesign --force --deep --options runtime --timestamp \
            --sign "$IDENTITY" \
            --keychain "$KEYCHAIN_PATH" \
            "$APP_PATH"

      - name: Verify signature
        run: |
          codesign --verify --deep --strict --verbose=2 "$APP_PATH"
          codesign -dvv "$APP_PATH" 2>&1 | grep -E 'TeamIdentifier|Authority' || true

      - name: Decode App Store Connect API key
        env:
          API_KEY_P8: ${{ secrets.APPSTORE_CONNECT_API_KEY_P8 }}
        run: |
          echo "$API_KEY_P8" | base64 --decode > /tmp/key.p8
          chmod 600 /tmp/key.p8

      - name: Zip for notarytool submission
        run: |
          ditto -c -k --keepParent "$APP_PATH" "DevTray-${{ steps.ver.outputs.short }}-notary.zip"

      - name: Notarize (wait, log on failure)
        env:
          KEY_ID: ${{ secrets.APPSTORE_CONNECT_KEY_ID }}
          ISSUER: ${{ secrets.APPSTORE_CONNECT_ISSUER_ID }}
        run: |
          set -e
          SUBMISSION_JSON=$(xcrun notarytool submit \
            "DevTray-${{ steps.ver.outputs.short }}-notary.zip" \
            --key /tmp/key.p8 \
            --key-id "$KEY_ID" \
            --issuer "$ISSUER" \
            --wait \
            --output-format json)
          echo "$SUBMISSION_JSON"
          STATUS=$(echo "$SUBMISSION_JSON" | jq -r '.status')
          SUBMISSION_ID=$(echo "$SUBMISSION_JSON" | jq -r '.id')
          if [ "$STATUS" != "Accepted" ]; then
            echo "::error::Notarization failed: status=$STATUS, submission=$SUBMISSION_ID"
            xcrun notarytool log "$SUBMISSION_ID" \
              --key /tmp/key.p8 \
              --key-id "$KEY_ID" \
              --issuer "$ISSUER"
            exit 1
          fi

      - name: Staple ticket
        run: |
          xcrun stapler staple "$APP_PATH"
          xcrun stapler validate "$APP_PATH"

      - name: Zip stapled app (final shipping artifact)
        id: zip
        run: |
          ZIP="DevTray-${{ steps.ver.outputs.short }}.zip"
          ditto -c -k --keepParent "$APP_PATH" "$ZIP"
          echo "zip=$ZIP" >> "$GITHUB_OUTPUT"
          echo "length=$(stat -f%z "$ZIP")" >> "$GITHUB_OUTPUT"
```

Notes for the implementer:
- Indentation is **6 spaces** for `- name:` (steps are inside a job's `steps:` array). Match the surrounding v0.8 steps exactly — `cat -A` to spot tab/space drift if anything looks wrong.
- The final step (`Zip stapled app`) inherits the `id: zip` and outputs `zip` + `length` — drop-in replacement for the v0.8 Zip step. The subsequent "EdDSA-sign", "Create GitHub Release", and "Update appcast" steps reference `${{ steps.zip.outputs.zip }}` / `${{ steps.zip.outputs.length }}` and require no edits.
