# DevTray v0.11 — License validation + paywall + LemonSqueezy MoR · Design

**Date:** 2026-05-30
**Cycle:** v0.11 (turns the v0.10 freeware-but-signed product into a paid product). Ships as `0.11.0` / build `11`.
**Branch:** `feat/v0.11-paywall` (in place, no worktree — backend deploy + secret setup run better on the main checkout, same as v0.8/v0.10).
**Predecessor:** v0.10 spec `docs/superpowers/specs/2026-05-29-devtray-v0.10-notarization-design.md` (latest format). v0.9 spec `docs/superpowers/specs/2026-05-29-devtray-v0.9-complete-v1-design.md` (pattern for new packages + new Settings tab).
**Reference:** roadmap memory `devtray-roadmap.md` (license/paywall was always the last v0.x blocker before v1.0 launch).

## Goal

v0.10 shipped a real Developer-ID-signed, Apple-notarized, stapled DevTray — Gatekeeper friction is dead, the binary is trustworthy. **But there's no way to charge for it.** No license validation, no trial, no paywall, no checkout. DevTray today is freeware-but-polished.

This cycle adds the perna that monetizes:

1. **EdDSA-signed license keys** in JWT format. Signed once by our backend, verified offline forever by every running copy of DevTray using a pubkey embedded in `Info.plist`. Same crypto primitive as Sparkle's update signing, separate keypair.
2. **LemonSqueezy as Merchant of Record.** They run the checkout (Stripe + Apple Pay + others), collect/remit VAT/sales tax worldwide, handle refunds + fraud + chargebacks, and fire a webhook to our backend on `order_created` / `order_refunded`. Flat fee ~5% + 50¢/txn; massively cheaper than incorporating in N tax jurisdictions ourselves.
3. **Cloudflare Workers backend** — four routes (`/webhook`, `/activate`, `/deactivate`, `/status`), two KV namespaces, three secrets. Free tier covers us for years.
4. **Trial-then-paywall app behavior** — 14-day hard-expiry trial, then most-tools-stay-free / Snippets-save-gated read-only mode (cluster 1 decision).

End result: someone lands on `devtray.app`, clicks Buy, pays $19, gets an email with a license key + `devtray://activate?…` deep link, opens DevTray, license is verified locally, activation count incremented on the backend, app moves from `.trialing` (or `.trialExpired`) to `.licensed`. Auto-update via Sparkle continues to work for everyone regardless of license state.

**Not a Swift-only cycle.** Adds a TypeScript backend in `backend/license/`. Execution uses `superpowers:subagent-driven-development` with separate subagents per language (Swift / TS) to keep context tight.

## Brainstorm decisions

| Question | Decision | Why |
|---|---|---|
| **Pricing model** | **One-time, paid upgrade for v2.x** (Sublime/Tower model) | Aligns with dev-tool buyer expectation. Subscription rejected: cancel friction + recurring billing infra DevTray doesn't need (no SaaS marginal cost). |
| **Price (v1.x)** | **$19 USD one-time** | Benchmarks: Tower $89, Paw $49, Charles $50. DevTray scope sits below them. $19 = psychological middle (not "cheap", not "premium"). |
| **Trial duration** | **14 days, hard expiry, no credit card upfront** | 14 = Sublime/JetBrains convention. Hard expiry (vs nag-only) actually drives conversion. No CC = critical for dev-tool friction. |
| **Post-trial free tier** | **"Mostly free" — all 13 tools + Spotlight + smart-paste + frequent-tools stay free forever; Snippets save/edit/delete + Settings → Tools toggles are the paywall surface** | Tools work for free indefinitely → goodwill + warm purchase intent. Snippets is the one feature with persistent user data → strongest hostage. Settings → Tools = "personalization" which is also a paid feel. |
| **Multi-machine** | **3 activations per license, server-tracked, deactivable** | 1 = frustrating (laptop + work Mac + Mac mini is a common dev setup). 5+ = invites sharing. 3 = Sublime/Tower/Paw convention. |
| **License delivery** | **Email with both: (a) bare paste-able key, (b) `devtray://activate?license=…` deep link** | Email + paste is robust to URL-scheme-stripping mail clients; deep link is the one-click happy path. Costs ~4 lines of Info.plist + an `onOpenURL` handler. |
| **Offline vs online validation** | **Offline-first EdDSA + best-effort online heartbeat (launch + every 7 days)** | EdDSA local verify means airplane usage works. Heartbeat is the revoke channel. Network failure → silently treat as no-op (never lock people out for flaky wifi). Only an explicit `revoked:true` from server flips state. |
| **License key format** | **JWT** with `DT1-` prefix for schema versioning | Standard JWS (debuggable at jwt.io + via DevTray's own JWT tool). `DT1-` prefix lets future `DT2-` change claim shape without breaking decoder. Hardcoded `alg=EdDSA` check in validator blocks `alg:none` confusion attacks. |
| **Backend hosting + language** | **Cloudflare Workers + TypeScript** (`@noble/ed25519` + `hono` router) | Free tier covers 100k req/day, no cold starts, global edge, deploy via Wrangler in seconds. TS + `@noble/ed25519` = zero-deps pure JS EdDSA. |
| **Domain** | **Buy `devtray.app` via Cloudflare Registrar** (~$15/year) | Doubles as the eventual launch landing-page domain. Cloudflare Registrar auto-configures DNS for the Workers route. Backend at `api.devtray.app/…`. |
| **EdDSA keypair separation from Sparkle's** | **Yes — fresh dedicated keypair for licenses** | Sparkle private key lives in `SPARKLE_PRIVATE_KEY` GitHub secret + author's Keychain; license private key lives in CF Workers secret only. Mixing = single-point-of-failure. |
| **Machine identifier** | **`IOPlatformUUID` via IOKit, hashed `SHA256(uuid \|\| license_uuid)` before sending to backend** | UUID is persistent + hardware-bound + survives wipe/reinstall (mobo swap excepted). Hashing it before send = privacy posture, server never sees raw hardware fingerprint. |
| **Activation flow** | **Only on initial activation (`POST /activate`). After that, local EdDSA verify only. Heartbeat is a separate path with different semantics.** | Minimal server load. JWT proves ownership; activation count is server-side state. Re-pasting the same JWT on a known-good machine is a no-op against the activation set. |
| **Refund** | **30-day no-questions-asked (LS default)** | LS handles the buyer-facing refund UX. Webhook `order_refunded` → mark `revoked:true` → next heartbeat flips app state. |
| **Bundle rename** | **Cosmetic only — `PRODUCT_NAME: DevTray` (so `.app` file shows as `DevTray.app`), keep target name `DevTrayApp` so bundle ID stays `com.devtray.DevTrayApp`** | Full rename = Sparkle bundle-ID consistency break (existing v0.10 testers need manual redownload — same friction as v0.8→v0.10). Cosmetic rename = zero Sparkle impact, ~5 minutes of work, gets the visible "DevTray.app on disk" win. |
| **CF KV vs Durable Objects** | **KV** | Eventual consistency (~60s) means worst-case race is "user activates 2 machines in <60s, both pass, gets 4 slots on a 3-license" — theoretical at scale 0. Migrate to DO if abuse appears. |
| **Webhook events to handle** | **`order_created` (mint) + `order_refunded` (revoke). Nothing else.** | One-time digital purchase → `order_created` = payment captured. `subscription_*` doesn't apply. |
| **Webhook signature verification** | **Mandatory HMAC-SHA256 verify on every webhook using `LEMONSQUEEZY_WEBHOOK_SECRET`.** | Without it, anyone can POST `order_created` to our endpoint and mint themselves a free license. |
| **Test mode isolation** | **Inspect `meta.test_mode` boolean on every webhook payload. If `true`, route to a separate `LICENSES_TEST` KV namespace (or refuse outright depending on env).** | Without isolation, smoke testing accidentally seeds prod KV with junk licenses. |
| **Webhook idempotency** | **Store `event_id → processed_at` in `EVENTS` KV namespace, TTL 7d. Skip duplicates.** | LS retries failed webhooks. Without idempotency, retries mint duplicate licenses. |
| **Auto-update vs license state** | **Never gated. Sparkle runs independently of `LicenseValidator`. Expired-trial users still get v0.12.** | Anti-pattern to gate updates behind paywall; Sparkle has no visibility into license state anyway. |
| **Email provider** | **Resend** (free tier 3k/month) | Simplest API + good DX. `RESEND_API_KEY` as Worker secret. Domain verification on `devtray.app` (one-time DNS work). |
| **LemonSqueezy product config** | **Store=`DevTray`, Product=`DevTray`, Variant=`License — v1.x`. LS's own license-keys feature DISABLED (we generate our own EdDSA JWTs).** | Variant name leaves room for v2.x upgrade SKU later. LS-managed licenses would be online-only, which contradicts our offline-first decision. |
| **Self-service deactivate-from-broken-Mac portal** | **NOT built in v0.11. Email-driven support (`support@devtray.app`) — author manually hits an admin endpoint.** | At <100 customers, manual support is fine. A self-service portal needs auth + UI + magic-link email = big scope creep. Re-evaluate at first launch milestone or first abuse report. |
| **Grandfather strategy for v0.10 testers** | **Treat them as new trial users — 14 days from first v0.11 launch.** | Tester pool is tiny (≤10 people). Goodwill is better delivered out-of-band ("here's a license, thanks for testing") than via a detection mechanism that costs code to ship. |
| **Logging/observability** | **CF Workers `console.log` → CF dashboard. Capture `event_id` + `license_uuid` + outcome on every webhook + activation. No Sentry/Logflare for v0.11.** | Sufficient for the traffic level. Revisit at first abuse incident or first 100 customers. |
| **Spec scope = one cycle** | **Same repo (`backend/license/`), one PR, two subagent tracks at execution (Swift + TS)** | Coherent single-feature scope. Nothing ships until both sides wire. Subagent split is execution discipline, not project scoping. |

## Architecture decisions (not brainstorm — follow from above)

- **JWT verifier hardcodes `alg=EdDSA`.** Any other `alg` value → reject. No JWK lookup, no key-id rotation in v0.11 (single embedded pubkey).
- **License pubkey lives in `Info.plist`** as `LICENSE_PUBLIC_KEY` (base64-encoded raw 32 bytes — Ed25519 pubkey). Read once at launch by `LicenseValidator.shared`. Guard: if missing or unparseable, app refuses to start with a fatal banner ("license subsystem misconfigured"). Same fail-loud posture as v0.8's `SUPublicEDKey` placeholder guard.
- **`LicenseStore` Keychain service = `com.devtray.DevTrayApp.license`** (matches the bundle ID we're keeping stable). Items: license JWT string, ISO8601 activation timestamp, hashed machine UUID, trial start, last_seen_at. All under one service, each as a distinct account key.
- **`TrialTracker` writes to Keychain, NOT UserDefaults.** UserDefaults is user-resettable via `defaults delete`; Keychain isn't (unless they wipe the entire login Keychain, which destroys lots of other data — high friction). Tampering detection: `last_seen_at` written on every launch. If `now < last_seen_at - 1h` (1h grace for legit clock corrections), TrialTracker enters `.trialExpired` immediately.
- **`MachineIdentifier` reads `IOPlatformUUID` once per process** and caches. The hash sent to backend is `SHA256("v1:" || license_uuid || ":" || raw_uuid)`. The `"v1:"` prefix reserves the option to change hashing later (different salt for v2 licenses) without colliding with v1 records.
- **`LicenseClient` uses `URLSession` with a 3-second timeout** for heartbeat (silently fails). Activation calls use a 10-second timeout (hard-fails with user-visible error). All bodies are JSON; all responses are JSON.
- **`LicenseState` is the single source of truth** consumed by gating views. Published into `@Environment` via a `LicenseStateKey`. Re-derived by `LicenseValidator` on launch + on heartbeat response + on activation/deactivation result.
- **No license-state in the registry.** Tool views don't know about licensing. Only `SnippetEditorView`, `SnippetsListView` (toolbar import/export), and `ToolsPreferencesTab` (Settings → Tools) read `LicenseState` and route to PaywallView when gated.
- **Single SwiftUI view modifier `.licensed { … } else: { PaywallView() }`** as the gating primitive. Three call sites (above). No ad-hoc `if state == .licensed { … }` scattered around.
- **Backend is stateless except for KV.** No in-memory caches, no rate limiting state in v0.11 (CF Workers gives some at the network edge). Add explicit rate-limit middleware if abuse appears.
- **Backend never logs JWTs or raw machine UUIDs.** Logs include `event_id`, `license_uuid`, `machine_hash` (already hashed), outcome, timestamp. PII (email) only on `/webhook` audit lines.
- **JSON casing convention** — backend emits `snake_case` field names (TS / hono / LemonSqueezy native style). Swift decodes with `JSONDecoder().keyDecodingStrategy = .convertFromSnakeCase` so Swift structs use idiomatic `camelCase`. Encoding direction (Swift → backend) uses the matching `.convertToSnakeCase` encoder. `LicenseClient` centralizes one configured `JSONDecoder` + `JSONEncoder` instance; nobody else constructs these.
- **Sparkle and LicenseValidator are independent subsystems.** Neither imports the other. `UpdaterController` (v0.8) keeps running regardless of `LicenseState`.

## Components

### 1. `Packages/DevTrayCore/Sources/DevTrayCore/Licensing/` (new module)

Six files, no external deps beyond `CryptoKit` + `IOKit` (both Apple frameworks):

#### `LicenseValidator.swift`

```swift
public actor LicenseValidator {
    public static let shared = LicenseValidator()
    private let publicKey: Curve25519.Signing.PublicKey

    public init() {
        // Read LICENSE_PUBLIC_KEY from Info.plist; fatal if missing/unparseable.
    }

    public func verify(_ rawKey: String) throws -> LicenseClaims {
        // 1. Strip "DT1-" prefix (else throw .unsupportedSchema).
        // 2. Split on "." into 3 segments (header, payload, sig).
        // 3. Base64url-decode header; assert alg == "EdDSA" + typ == "JWT".
        // 4. Reconstruct signing input = header + "." + payload.
        // 5. publicKey.isValidSignature(sig, for: signingInput) or throw .invalidSignature.
        // 6. Decode payload JSON → LicenseClaims (license_uuid, email, iat, tier).
        // 7. Assert tier == "v1" (forward-compat — v2 licenses use DT2- prefix).
        // 8. Return claims.
    }
}

public struct LicenseClaims: Codable, Sendable, Equatable {
    public let licenseUUID: UUID  // sub
    public let email: String       // email
    public let issuedAt: Date      // iat (unix seconds → Date)
    public let tier: String        // "v1"
}

public enum LicenseValidationError: Error {
    case unsupportedSchema     // DT2-, DT0-, no prefix
    case malformedToken        // not 3 dot-segments
    case invalidAlgorithm      // alg != EdDSA, typ != JWT
    case invalidSignature
    case malformedPayload      // JSON decode failed
    case unsupportedTier       // tier != "v1"
}
```

Tests (`LicensingTests/LicenseValidatorTests.swift`): happy path round-trip (signs a JWT with a test private key, verifies with paired pubkey); rejects each error case explicitly; rejects `alg:none` token.

#### `LicenseStore.swift`

```swift
public actor LicenseStore {
    public init(keychain: KeychainProtocol = SystemKeychain())

    public func storeLicense(_ jwt: String, claims: LicenseClaims) async throws
    public func storedLicense() async throws -> (jwt: String, claims: LicenseClaims)?
    public func clearLicense() async throws

    public func storeMachineHash(_ hash: String) async throws
    public func storedMachineHash() async throws -> String?
}

protocol KeychainProtocol: Sendable {
    func set(_ data: Data, account: String) throws
    func get(account: String) throws -> Data?
    func delete(account: String) throws
}
```

All items live under service `com.devtray.DevTrayApp.license`. Accounts: `license_jwt`, `license_claims_json`, `machine_hash`, `activated_at`, `trial_start`, `last_seen_at`. `KeychainProtocol` lets tests inject a `InMemoryKeychain` fake.

#### `LicenseState.swift`

```swift
public enum LicenseState: Equatable, Sendable {
    case untrialed
    case trialing(daysLeft: Int)
    case trialExpired
    case licensed(claims: LicenseClaims)
    case revoked
}
```

A pure-data enum. Computed by `LicenseValidator.computeState(now:)` on launch + on heartbeat result + on activation/deactivation result.

#### `TrialTracker.swift`

```swift
public actor TrialTracker {
    public init(store: LicenseStore, clock: @escaping () -> Date = Date.init)

    /// Returns the trial start; bootstraps on first call.
    public func ensureTrialStarted() async throws -> Date

    /// Returns (daysLeft, expired) for the 14-day window from trial_start.
    public func currentStatus() async throws -> (daysLeft: Int, expired: Bool)

    /// Returns true if clock setback exceeds 1h grace.
    public func detectClockSetback() async throws -> Bool

    /// Writes current `clock()` to last_seen_at.
    public func recordLastSeen() async throws
}
```

Trial window = 14 days hard. `currentStatus()` clamps `daysLeft >= 0`. `detectClockSetback` reads `last_seen_at`; if `clock() < last_seen_at - 3600`, returns true → caller forces `.trialExpired`.

#### `MachineIdentifier.swift`

```swift
public enum MachineIdentifier {
    /// Reads IOPlatformUUID via IOKit. Cached after first call.
    public static func raw() throws -> String

    /// SHA256("v1:" + licenseUUID + ":" + raw()) as lowercase hex.
    public static func hash(for licenseUUID: UUID) throws -> String
}
```

Implementation uses `IOServiceMatching("IOPlatformExpertDevice")` + `IORegistryEntryCreateCFProperty(_:, kIOPlatformUUIDKey as CFString, ...)`. Throws `.unavailable` if either fails (CI runners without IORegistry).

Unit-testable via a `MachineIDProvider` protocol fake; the public static enum is the production wiring.

#### `LicenseClient.swift`

```swift
public actor LicenseClient {
    public init(baseURL: URL = URL(string: "https://api.devtray.app")!,
                session: URLSession = .shared)

    public func activate(licenseUUID: UUID, machineHash: String) async throws -> ActivationResponse
    public func deactivate(licenseUUID: UUID, machineHash: String) async throws
    public func heartbeat(licenseUUID: UUID, machineHash: String) async throws -> HeartbeatResponse
}

public struct ActivationResponse: Codable, Sendable {
    public let ok: Bool
    public let activationsRemaining: Int  // e.g. 2 if this brought us to 1/3, 0 if at cap
}

public struct HeartbeatResponse: Codable, Sendable {
    public let revoked: Bool
}
```

`activate` is 10-second timeout, hard error on network failure (caller shows UI). `heartbeat` is 3-second timeout, swallows network errors and returns `.revoked = false` (the safe default — we never lock out for flaky wifi). `deactivate` is 10-second timeout, surfaces errors to user.

### 2. `Packages/DevTrayUI/Sources/DevTrayUI/Paywall/` (new submodule)

Three SwiftUI files + one view modifier:

#### `PaywallView.swift`

Modal sheet shown when a gated action fires while `LicenseState != .licensed`. Primary CTA: "Buy License — $19" → opens `https://devtray.app/buy` (which redirects to the LS checkout URL). Secondary: "Activate existing license" → focuses LicenseTab. Tertiary: "Continue trial" — only shown when `state == .trialing`; dismisses sheet.

Layout: app icon + headline + 3-row feature comparison (Trial vs Licensed vs Free) + CTAs. Lazy-loaded; doesn't ship in PopoverRoot's main render tree.

#### `LicenseTab.swift`

Settings tab (sits alongside General/Shortcuts/Tools/About). Sections:

- **Status:** current `LicenseState` rendered as one of 4 cards (Untrialed/Trialing/Expired/Licensed/Revoked). Shows email + license_uuid prefix + activation date for `.licensed`.
- **Activate:** TextField for pasting `DT1-…` key + Activate button (disabled if state is `.licensed`).
- **Deactivate this Mac:** button, only shown when `.licensed`. Confirmation alert ("This frees up one of your 3 activation slots — are you sure?"). Calls `LicenseClient.deactivate` then `LicenseStore.clearLicense()`.
- **Buy a license:** link to `https://devtray.app/buy`, only shown when not `.licensed`.
- **Help link:** "Need to deactivate another Mac you can't reach? Email support@devtray.app" — only shown when `.licensed` or `.trialExpired`.

#### `TrialBanner.swift`

Persistent banner inside `PopoverRoot`'s top edge while `state == .trialing(daysLeft:)`. One line: "DevTray trial — N days left. [Buy License]". The CTA opens the buy URL directly (no PaywallView intermediary — it's already 1 click). Banner is hidden in all other states.

#### `PaywalledModifier.swift`

```swift
public extension View {
    /// Gates the modified view behind LicenseState (read from @Environment).
    ///
    /// State → behavior:
    ///   .untrialed, .trialing, .licensed → content rendered unchanged
    ///   .trialExpired, .revoked          → content rendered dimmed (opacity 0.4)
    ///                                      with a "🔒 Buy to unlock" overlay;
    ///                                      tap anywhere on the dimmed area opens PaywallView
    ///                                      (modal sheet); underlying content is non-interactive
    ///                                      while in this state.
    func paywalled() -> some View
}
```

No closure parameter. Single canonical modifier; the "paywall surface" decision is made once in the modifier impl, not at each call site. Call sites: `.paywalled()` after the relevant subview.

#### Route contracts

```
POST /webhook
  Headers: X-Signature: <hex HMAC-SHA256 of body using LEMONSQUEEZY_WEBHOOK_SECRET>
  Body: LemonSqueezy event JSON
  200 OK on success (must be 2xx or LS retries forever)
  401 if HMAC fails
  Idempotent: same event_id → no-op + log skip

POST /activate
  Body: { license_jwt: string, machine_hash: string }
  200 { ok: true, activations_remaining: number }
  400 if JWT malformed / wrong tier
  403 { error: "too_many_activations" } if at cap
  403 { error: "revoked" } if license is revoked
  404 if license_uuid not in LICENSES KV

POST /deactivate
  Body: { license_jwt: string, machine_hash: string }
  200 { ok: true }
  404 if license unknown or machine_hash not in activations[]

GET /status?license=<uuid>&machine=<hash>
  200 { revoked: boolean }   (always 200 if request is well-formed;
                              `revoked:true` covers all "no longer valid" cases:
                              license unknown, license revoked, machine not in activations[])
  400 if query params malformed
```

#### KV schema

**Namespace `LICENSES`** (prod) / `LICENSES_TEST` (test mode):

```ts
// key = license_uuid (string)
type LicenseRecord = {
  user_email: string;
  created_at: number;           // unix seconds
  activations: Activation[];
  revoked: boolean;
  test_mode: boolean;
  ls_order_id: string;          // LemonSqueezy order id (audit)
};

type Activation = {
  machine_hash: string;
  activated_at: number;         // unix seconds
};
```

**Namespace `EVENTS`** (single namespace, shared):

```ts
// key = ls_event_id (string), TTL 7 days
type EventRecord = {
  processed_at: number;
  outcome: "minted" | "revoked" | "skipped";
};
```

```
  200 → activations[] cleared
```

Used manually when a buyer emails `support@devtray.app` to release a Mac they can't reach. Not used by the app.

## License JWT format

```
DT1-<base64url(header)>.<base64url(payload)>.<base64url(signature)>

header  = {"alg":"EdDSA","typ":"JWT"}
payload = {
  "iss": "api.devtray.app",
  "sub": "<license_uuid>",        // UUID v4, also key in LICENSES KV
  "email": "buyer@example.com",
  "iat": 1748560000,              // unix seconds
  "tier": "v1"
}
signature = Ed25519(base64url(header) + "." + base64url(payload),
                   privateKey = LICENSE_PRIVATE_KEY)
```

- No `exp` claim. License perpetual for tier; revocation is server-side state in KV, surfaced via heartbeat.
- No machine fingerprint in payload. JWT proves ownership; activation count is server-side.
- `DT1-` prefix is OUTSIDE the JWT (versioning is OURS, not JOSE's).
- Validator must enforce `alg == "EdDSA"` BEFORE any signature operation (blocks `alg:none` attack).
- Validator must enforce `tier == "v1"` (forward-compat — DT2 licenses targeting v2.x will have `tier:"v2"` and be rejected by DT1-decoder).

## Trial state machine

```
                 ┌──────────────┐
   first launch  │  .untrialed  │  (no Keychain entry yet)
                 └──────┬───────┘
                        │ TrialTracker.ensureTrialStarted writes trial_start = now
                        ▼
            ┌─────────────────────────┐
            │  .trialing(daysLeft:N)  │  N = clamp(14 - days_since(trial_start), 0...14)
            └──┬────────────────┬─────┘
   user activates                │ N reaches 0 OR clock setback detected
   valid license │                ▼
                 │      ┌────────────────────┐
                 │      │   .trialExpired    │
                 │      └────────┬───────────┘
                 │               │ user activates valid license
                 ▼               ▼
            ┌────────────────────────┐    heartbeat returns revoked:true
            │   .licensed(claims)    │ ───────────────────────────────► .revoked
            └────────────────────────┘                                  │
                          ▲                                             │ user activates
                          └─────────────────────────────────────────────┘  a different valid license
```

Tampering: every launch + every state recompute writes `last_seen_at`. If `clock() < last_seen_at - 1h` → force `.trialExpired` regardless of `trial_start`. Once `.licensed`, clock setback is ignored (you paid; we don't re-trial you).

## Data flow scenarios

### Scenario 1 — Purchase → email → activation

1. User clicks "Buy License" on `devtray.app` (or in app's PaywallView / LicenseTab) → redirects to LS checkout URL.
2. User pays $19. LS captures payment.
3. LS POSTs `order_created` webhook (HMAC-signed) to `https://api.devtray.app/webhook`.
4. Backend `routes/webhook.ts`:
   - Verifies HMAC against `LEMONSQUEEZY_WEBHOOK_SECRET` → 401 if invalid.
   - Reads `meta.event_id` → check `EVENTS` KV for duplicate → 200 + log skip if seen.
   - Reads `meta.test_mode` → routes to `LICENSES_TEST` if true (or rejects, env-gated).
   - Generates `license_uuid = crypto.randomUUID()`.
   - Builds payload `{ iss, sub: license_uuid, email, iat, tier: "v1" }`.
   - Signs JWT with `LICENSE_PRIVATE_KEY` → builds `DT1-<jwt>` string.
   - Writes `LICENSES[license_uuid] = { user_email, created_at, activations: [], revoked: false, test_mode, ls_order_id }`.
   - Writes `EVENTS[event_id] = { processed_at, outcome: "minted" }` with TTL 7d.
   - Sends email via Resend: subject "Your DevTray license", body contains bare `DT1-…` key + `devtray://activate?license=DT1-…` deep link.
   - Returns 200.
5. User opens email, clicks deep link.
6. macOS dispatches to DevTray via URL scheme; `DevTrayApp.onOpenURL` handler:
   - Parses `?license=` query param.
   - Calls `LicenseValidator.verify(jwt)` → gets `LicenseClaims` or shows error.
   - Calls `MachineIdentifier.hash(for: claims.licenseUUID)`.
   - Calls `LicenseClient.activate(licenseUUID, machineHash)`.
7. Backend `routes/activate.ts`:
   - Reads `LICENSES[license_uuid]` → 404 if missing.
   - Check `revoked` → 403 if true.
   - Check `activations.length >= 3` → 403 `too_many_activations` if at cap (and `machine_hash` not already present).
   - If `machine_hash` already in `activations[]` → 200 + no-op (re-activation on same machine is silent).
   - Else append `{ machine_hash, activated_at: now }` → write back → 200 `{ ok: true, activations_remaining: 3 - len }`.
8. App: `LicenseStore.storeLicense(jwt, claims)` + `storeMachineHash(hash)` → `LicenseState = .licensed(claims)` → UI re-renders.

### Scenario 2 — Heartbeat (background, every 7 days while running)

```
DevTray.app                              Backend
  Timer fires every 7 days,
  also on app launch if .licensed
  │
  │ GET /status?license=<uuid>&machine=<hash>  (timeout 3s)
  │ ──────────────────────────────────────────►│
  │                                            │ read LICENSES[uuid]
  │                                            │   KV miss             → 200 {revoked: true}
  │                                            │   revoked=true        → 200 {revoked: true}
  │                                            │   machine not in acts → 200 {revoked: true}  (stolen JWT)
  │                                            │   else                → 200 {revoked: false}
  │◄───────────────────────────────────────────│
  │
  │ network error / timeout → swallow, no state change
  │ revoked:true → clearLicense() → LicenseState = .revoked
  │ revoked:false → no-op
```

The "machine_hash not in activations" branch handles the case where a JWT is stolen and pasted into a 4th machine that succeeds offline (EdDSA verifies fine) but is detected on first heartbeat — server doesn't know that machine, returns revoked.

### Scenario 3 — Refund → revoke

1. Buyer requests refund in LS portal (or author processes it).
2. LS approves → POSTs `order_refunded` webhook (HMAC-signed).
3. Backend `routes/webhook.ts`:
   - Verify HMAC, idempotency.
   - Read `meta.order_id` → look up `LICENSES` entries with matching `ls_order_id`.
   - Set `revoked = true` → write back.
   - No email sent (LS handles refund-confirmation email).
4. Up to 7 days later (or on next launch), app heartbeat returns `revoked:true` → state → `.revoked` → next gated action triggers PaywallView.

### Scenario 4 — User deactivates this Mac

1. User clicks "Deactivate this Mac" in LicenseTab → confirmation alert.
2. App: `LicenseClient.deactivate(licenseUUID, machineHash)`.
3. Backend `routes/deactivate.ts`:
   - Read `LICENSES[license_uuid]` → 404 if missing.
   - Filter out matching `machine_hash` from `activations[]` → write back.
   - 200 `{ ok: true }`.
4. App: `LicenseStore.clearLicense()` → `LicenseState` recomputed from trial state → if still in trial window, `.trialing(daysLeft:N)`; if past 14 days, `.trialExpired`.

### Scenario 5 — Email-driven release-all (manual support)

1. Buyer emails `support@devtray.app`: "I lost my old Mac and used 3/3 slots."
2. Author verifies via LS order history, then runs:
   ```bash
   curl -X POST https://api.devtray.app/admin/release-all \
     -H "X-Admin-Token: $ADMIN_TOKEN" \
     -d '{"license_uuid":"..."}'
   ```
3. Backend clears `activations[]` → buyer can re-activate on new Mac.
4. Author replies to email confirming.
