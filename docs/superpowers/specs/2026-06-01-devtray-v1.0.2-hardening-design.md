# DevTray v1.0.2 — Hardening + polish · Design

**Date:** 2026-06-01
**Cycle:** v1.0.2 (second patch on top of v1.0 — bundles tech-debt cleanup + branding polish). Ships as `1.0.2` / build `14`.
**Branch:** `feat/v1.0.2-hardening` (in place, no worktree — backend deploy + author-only secrets run better on the main checkout).
**Predecessors:**
- v1.0 spec: `docs/superpowers/specs/2026-05-31-devtray-v1.0-paddle-launch-design.md`
- v1.0.1 spec: `docs/superpowers/specs/2026-06-01-devtray-v1.0.1-refund-detection-design.md`
**Trigger:** Two distinct gaps remain after v1.0.1 shipped: (a) the mint path still has the retroactive-mint race that produced the `3f365dd7-…` zombie license in v1.0 (a refund that finalized between Paddle dispatch and our processing would still mint a valid license); (b) contract drift between Swift and TS has produced 9 cumulative bugs over the last two cycles with no e2e gate that would catch them before smoke. Bundling branding polish (custom icon + hero GIF) keeps the public surface from looking placeholder-y while we ship the hardening.

## Goal

v1.0.1 closed the refund detection gap *after* mint via webhook + cron polling. v1.0.2 closes the gap *before* mint by checking transaction status on the critical path of `mintLicense`, so a refund that finalized in-flight produces a born-revoked record instead of a valid license that has to be revoked later. It also adds the first end-to-end test infrastructure (`scripts/e2e-webhook.sh` + Worker-side test-mode routing) so future cycles catch Swift↔TS contract drift before smoke, removes the `ls_order_id` LemonSqueezy-era fallback that has had zero production usage for two cycles, and adds Tier C branding polish (custom icon via Fiverr/Upwork + hero GIF) so the public surface is no longer placeholder.

**Issue 1 — Pre-mint race window (Tier A).** The v1.0 zombie license `3f365dd7-…` was minted from a transaction whose refund had already been issued in Paddle's dashboard but whose `adjustment.created` webhook had not yet reached our Worker when `transaction.completed` did. The handler had no way to know the money was already on its way back, so it minted a valid license and emailed it. v1.0.1's polling reconciliation catches this on the next 30-min sweep, but the email already went out — the customer has a key in their inbox that becomes invalid minutes later. The fix is defense-in-depth: before signing the JWT, GET the transaction from Paddle and confirm status is not already `refunded` or `partially_refunded`.

**Issue 2 — `reconcile.ts` KV list propagates errors (Tier A).** Line 26 of `reconcile.ts` (`const list = await env.LICENSES.list()`) is not wrapped. A KV outage during a cron sweep would crash the scheduled handler silently (logged via Workers observability but with no `ReconcileResult` JSON returned by the `/admin/reconcile` HTTP path). Wrap it so callers see `errors: 1`.

**Issue 3 — No e2e infra (Tier B).** v0.11 caught 4 Swift↔TS contract bugs at smoke time. v1.0 caught 5 more (transaction.refunded didn't exist, customer.email wasn't embedded, magic link path was wrong, `data.origin` wasn't a sandbox flag, `pending_approval` status wasn't gated). Each was a contract assumption that would have failed against the live API on a fixture-driven script. A bash script + Worker-side test-mode routing covers the webhook → KV path end-to-end without paying $14.99 per smoke.

**Issue 4 — `ls_order_id` fallback debt (Tier B).** The `ls_order_id` field on `LicenseRecord` and the `|| rec.ls_order_id === transactionId` fallback in `revokeByAdjustment` were transition-cycle artifacts from the LemonSqueezy → Paddle migration. The transition is complete; both should go if a prod KV audit confirms zero records carry `ls_order_id`.

**Issue 5 — Placeholder branding (Tier C).** The menubar uses SF Symbol `tray.full.fill` (acceptable placeholder), but Dock / About / DMG show the same generic glyph. `docs/hero.gif` is a broken link in README + landing. Both make the product look unfinished to first-time visitors.

**Not in scope (deferred to v1.1+):**
- Public announcement (6 channels: Twitter, LinkedIn, 4 Reddit subs) — drafts exist at `docs/launch/v1.0-announcement-drafts.md`; ship to v1.1 once polish lands.
- Full e2e (miniflare + Swift driver) — `lite` script covers the webhook contract this cycle. Full driver is v1.0.3.
- CI integration of `scripts/e2e-webhook.sh` — requires `wrangler dev --remote` in background + GH Actions secret wiring; v1.0.3.
- Pagination of KV `list()` past 1000 keys — still under threshold.
- Auto-unrevoke if a refund is later rejected (covered in v1.0.1 spec rationale).

## Architecture decisions (follow from above)

- **Pre-mint guard is on the critical path of `mintLicense`.** Adds one HTTPS round-trip to Paddle before signing. p50 latency budget: +~150ms. Acceptable — `transaction.completed` webhooks are not user-blocking (the buyer has already left the checkout page).
- **`fetchTransactionStatus` mirrors `fetchCustomerEmail`.** Same shape, same error handling, same fallback (`return null`). Pattern is already in the codebase and well-tested.
- **`reconcile.ts` try/catch is the only error-handling change.** Inner loop already has try/catch per-record; outer `list()` is the only unwrapped surface.
- **`ls_order_id` removal is type-system-driven.** Once the field is removed from `LicenseRecord`, TypeScript flags the `webhook.ts:184` fallback and the audit-trail comments in `types.ts:14-16` as remaining references. Fix-on-error path; no missed call sites.
- **E2E routing in `mintLicense` is one branch at the top.** If `transactionId.startsWith("txn_e2e_")`, the function skips `fetchTransactionStatus` (no Paddle call), skips `fetchCustomerEmail` (uses `e2e@devtray.app` stub), routes to `LICENSES_TEST` namespace via `licenseNamespace(env, true)`, and skips `sendLicenseEmail`. The rest of the function — sign JWT, build record, `putLicense`, `markEventProcessed` — runs identically against the test namespace. This means the script asserts the actual production code path minus the three external side-effects.
- **E2E routing in `revokeByAdjustment` is one branch at the top.** If `transactionId.startsWith("txn_e2e_")`, scan `LICENSES_TEST` instead of `LICENSES`. No other change.
- **`scripts/e2e-webhook.sh` is author-only this cycle.** It depends on `~/devtray-secrets/paddle_notification_secret.txt` for HMAC signing. CI integration is deferred.
- **The Worker-side test-mode branch sets `test_mode: true` on the LicenseRecord.** That field exists on the type already (it's been `false` for all real mints since v1.0); e2e mints flip it to `true` so any future query that filters by `test_mode === false` automatically excludes them.
- **App version bump is mechanical.** No Swift code changes for Tier A+B (backend-only). Tier C icon integration is a single `info.properties` entry + ICNS drop.
- **Custom icon workstream is async.** Agent work proceeds without blocking on icon delivery. If icon arrives before smoke → integrate + ship together. If icon arrives after smoke → tag `v1.0.2` without it, enqueue `v1.0.3` polish-only. Decision deferred to ship time.

**Add helper `fetchTransactionStatus`** (mirrors `fetchCustomerEmail` at lines 37-59):

```ts
async function fetchTransactionStatus(
  transactionId: string,
  env: Env,
  fetchImpl: FetchImpl,
): Promise<string | null> {
  const apiBase = env.PADDLE_API_BASE_URL || DEFAULT_API_BASE;
  try {
    const res = await fetchImpl(`${apiBase}/transactions/${encodeURIComponent(transactionId)}`, {
      method: "GET",
      headers: { Authorization: `Bearer ${env.PADDLE_API_KEY}` },
    });
    if (!res.ok) {
      const body = await res.text().catch(() => "<no body>");
      console.error(`[webhook] transaction fetch failed status=${res.status} txn=${transactionId} body=${body}`);
      return null;
    }
    const payload = (await res.json()) as { data?: { status?: string } };
    return payload?.data?.status ?? null;
  } catch (err) {
    console.error(`[webhook] transaction fetch error txn=${transactionId}`, err);
    return null;
  }
}
```

### 4. E2E test infra — `scripts/e2e-webhook.sh`

New file. Author-runnable bash script. Skeleton:

```bash
#!/usr/bin/env bash
set -euo pipefail

# Usage: ./scripts/e2e-webhook.sh
# Prereq: `wrangler dev --remote` running in another terminal (binds to localhost:8787),
#         and ~/devtray-secrets/paddle_notification_secret.txt readable.

WORKER_URL="${WORKER_URL:-http://localhost:8787}"
SECRET="$(cat ~/devtray-secrets/paddle_notification_secret.txt)"
TXN_ID="txn_e2e_$(uuidgen | tr 'A-Z' 'a-z' | tr -d '-' | head -c 16)"
EVENT_ID_MINT="evt_e2e_$(uuidgen | tr 'A-Z' 'a-z' | tr -d '-' | head -c 16)"
EVENT_ID_REFUND="evt_e2e_$(uuidgen | tr 'A-Z' 'a-z' | tr -d '-' | head -c 16)"

sign_and_post() {
  local body="$1"
  local ts="$(date +%s)"
  local mac
  mac=$(printf '%s:%s' "$ts" "$body" | openssl dgst -sha256 -hmac "$SECRET" -hex | awk '{print $NF}')
  curl -fsS -X POST "${WORKER_URL}/webhook" \
    -H "Content-Type: application/json" \
    -H "Paddle-Signature: ts=${ts};h1=${mac}" \
    --data "$body"
}

# Step 1: mint
mint_body=$(jq -nc --arg eid "$EVENT_ID_MINT" --arg txn "$TXN_ID" \
  '{event_id: $eid, event_type: "transaction.completed", data: {id: $txn, customer_id: "cus_e2e_stub"}}')
sign_and_post "$mint_body"
echo "[e2e] mint posted txn=$TXN_ID"

# Step 2: assert mint landed in LICENSES_TEST with revoked=false
# (poll wrangler kv key list LICENSES_TEST until a record with paddle_transaction_id=$TXN_ID appears)
# ... assertion logic ...

# Step 3: refund
refund_body=$(jq -nc --arg eid "$EVENT_ID_REFUND" --arg txn "$TXN_ID" --arg adj "adj_e2e_$(uuidgen)" \
  '{event_id: $eid, event_type: "adjustment.updated", data: {id: $adj, action: "refund", status: "approved", transaction_id: $txn}}')
sign_and_post "$refund_body"
echo "[e2e] refund posted txn=$TXN_ID"

# Step 4: assert record now has revoked=true
# ... assertion logic ...

# Step 5: cleanup — delete the LICENSES_TEST record + the two EVENTS records
# ... cleanup logic ...

echo "[e2e] pass"
```
