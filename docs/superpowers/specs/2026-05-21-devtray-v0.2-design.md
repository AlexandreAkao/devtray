# DevTray v0.2 — Design Document

**Date:** 2026-05-21
**Status:** Design (awaiting plan)
**Author:** Brainstorming session with user
**Predecessor:** [v0.1 foundation](2026-05-20-devtray-design.md) (shipped as `v0.1.0` on 2026-05-21)

## Summary

v0.2 adds four developer utility tools on top of the v0.1 foundation: **URL encode/decode**, **Hash** (MD5/SHA-1/SHA-256/SHA-512), **UUID/ULID generator** (v4/v7/ULID), and **Timestamp converter**. The work follows the architecture already established in v0.1 — every tool is a Swift package with a pure `XxxToolKit` engine plus a `XxxTool` SwiftUI view that conforms to the `Tool` protocol from `DevTrayCore`. No new dependencies are added; one small addition to `DevTrayCore` introduces a `.generators` category for the UUID tool.

## Goals

- Ship four new tools that feel native and respond instantly inside the existing popover.
- Reuse the v0.1 components (`CodeEditor`, `MonospaceOutput`, `InlineErrorBanner`) — no new `DevTrayUI` primitives.
- Keep tool engines pure and unit-testable without SwiftUI.
- No external dependencies. v4 UUIDs use `Foundation.UUID`; v7 and ULID are implemented in ~20 lines each using `SecRandomCopyBytes`; hashes use `CryptoKit`; timestamps use `Foundation.ISO8601DateFormatter`.

## Non-goals

- File input for the Hash tool (string only in v0.2; deferred).
- Timezone picker for the Timestamp tool (UTC + local only in v0.2; deferred).
- Component-level URL encoding modes (full-string percent-encoding via `.urlQueryAllowed` in v0.2).
- Custom Code Connect mappings, theming, or Spotlight integration (still v0.4+).

## Decisions (from brainstorming)

| # | Decision | Choice | Rationale |
|---|---|---|---|
| 1 | URL tool scope | Encode/Decode of full string with `.urlQueryAllowed` | Mirrors Base64Tool UX; covers 90% of dev-paste cases; component-level modes deferred to keep popover simple |
| 2 | Hash tool input | String only; output all 4 algorithms in hex simultaneously | File input is awkward inside a 420×540 popover; simultaneous output saves clicks |
| 3 | UUID/ULID formats | v4 + v7 + ULID, native implementation, no external deps | RFC 9562 v7 is short to implement; supply-chain risk minimized |
| 4 | UUID/ULID batch | Stepper 1–50, default 1, list of results with per-item Copy + Copy All | Common pattern in similar tools; 50 is enough for any realistic seeding |
| 5 | Timestamp input | Single field, auto-detect epoch s vs ms vs ISO 8601 | One field is faster than three; ambiguity solved by digit count (10=s, 13=ms) |
| 6 | Timestamp output | Always show epoch s, epoch ms, ISO 8601 UTC, ISO 8601 local | Covers the four shapes devs paste/copy daily |
| 7 | New category | Add `.generators` to `ToolCategory` | UUID/ULID don't fit `encoding`/`crypto`/`time`; small additive change |

## Tool details

### URL — `Packages/Tools/URLTool/`

**Engine (`URLToolKit`):**

```swift
public enum URLEngine {
    public static func encode(_ raw: String) -> Result<String, ToolError>
    public static func decode(_ raw: String) -> Result<String, ToolError>
}
```

- `encode`: `addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed)`. Empty input → `.invalidInput(reason: "Input is empty")`.
- `decode`: `removingPercentEncoding`. Returns `nil` for malformed → `.parseFailure(reason: "Invalid percent-encoding", hint: nil)`. Empty input → `.invalidInput`.

**View (`URLTool`):** Picker `Encode | Decode` + `CodeEditor` + `MonospaceOutput`. Mirrors `Base64ToolView` structurally.

**Tool conformance:**

```swift
public enum URLTool: Tool {
    public static let id: ToolID = "url"
    public static let displayName = "URL"
    public static let iconName = "link"
    public static let keywords = ["url", "uri", "percent", "encode", "decode"]
    public static let category: ToolCategory = .encoding
}
```

**Test cases:**
- `"hello world"` ↔ `"hello%20world"`
- `"café"` ↔ `"caf%C3%A9"`
- Decode `"%ZZ"` → `parseFailure`
- Empty input (either mode) → `invalidInput`

### Hash — `Packages/Tools/HashTool/`

**Engine (`HashToolKit`):**

```swift
public enum HashEngine {
    public static func md5(_ raw: String) -> Result<String, ToolError>
    public static func sha1(_ raw: String) -> Result<String, ToolError>
    public static func sha256(_ raw: String) -> Result<String, ToolError>
    public static func sha512(_ raw: String) -> Result<String, ToolError>
}
```

- Implemented via `CryptoKit.Insecure.MD5`, `Insecure.SHA1`, `SHA256`, `SHA512`.
- Output: lowercase hex string.
- Empty input → `.invalidInput`. UTF-8 conversion from Swift `String` cannot fail; no parse branch.

**View (`HashTool`):** `CodeEditor` for input + four `MonospaceOutput` rows labeled `MD5`, `SHA-1`, `SHA-256`, `SHA-512`. All four recompute on every keystroke.

**Tool conformance:**

```swift
public enum HashTool: Tool {
    public static let id: ToolID = "hash"
    public static let displayName = "Hash"
    public static let iconName = "lock.shield"
    public static let keywords = ["hash", "md5", "sha", "sha1", "sha256", "sha512", "digest"]
    public static let category: ToolCategory = .crypto
}
```

**Test cases:**
- MD5(`"hello"`) = `5d41402abc4b2a76b9719d911017c592`
- SHA-1(`"hello"`) = `aaf4c61ddcc5e8a2dabede0f3b482cd9aea9434d`
- SHA-256(`"hello"`) = `2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824`
- SHA-512(`"hello"`) — known vector
- SHA-256(`"héllo 🌍"`) — Unicode round-trip
- Empty input on each algorithm → `invalidInput`

### UUID/ULID — `Packages/Tools/UUIDTool/`

**Engine (`UUIDToolKit`):**

```swift
public enum IDFormat: String, CaseIterable, Sendable {
    case uuidV4
    case uuidV7
    case ulid

    public var displayName: String {
        switch self {
        case .uuidV4: return "UUID v4"
        case .uuidV7: return "UUID v7"
        case .ulid:   return "ULID"
        }
    }
}

public enum UUIDEngine {
    public static func generate(format: IDFormat, count: Int) -> Result<[String], ToolError>
}
```

- `count` validated to `1...50`; out of range → `.invalidInput(reason: "Count must be between 1 and 50")`.
- **v4:** `UUID().uuidString.lowercased()`.
- **v7 (RFC 9562):** 48-bit unix-ms timestamp + 4-bit version `0x7` + 12-bit rand + 2-bit variant `0b10` + 62-bit rand. Random bytes via `SecRandomCopyBytes`. Formatted as `XXXXXXXX-XXXX-7XXX-YXXX-XXXXXXXXXXXX` lowercase.
- **ULID:** 48-bit unix-ms timestamp + 80-bit random, Crockford Base32 encoded, 26 chars uppercase.

**View (`UUIDTool`):** Picker `UUID v4 | UUID v7 | ULID` + Stepper for count (1–50) + `Generate` button + scrollable list of results, each row = `MonospaceOutput` + Copy button. `Copy All` button copies joined-by-newline.

**Tool conformance:**

```swift
public enum UUIDTool: Tool {
    public static let id: ToolID = "uuid"
    public static let displayName = "UUID / ULID"
    public static let iconName = "number.square"
    public static let keywords = ["uuid", "ulid", "guid", "id", "generate", "v4", "v7"]
    public static let category: ToolCategory = .generators
}
```

**Test cases:**
- v4: matches regex `^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$`; batch of 10 produces 10 unique strings.
- v7: version nibble = `7`, variant top bits = `0b10`, timestamp prefix decodes to within ±1s of `Date().timeIntervalSince1970 * 1000`.
- ULID: 26 chars, alphabet `0123456789ABCDEFGHJKMNPQRSTVWXYZ` only (Crockford), two ULIDs generated in the same millisecond share the timestamp prefix.
- `count = 0`, `count = 51`, `count = -1` → `invalidInput`.

### Timestamp — `Packages/Tools/TimestampTool/`

**Engine (`TimestampToolKit`):**

```swift
public struct TimestampBreakdown: Equatable, Sendable {
    public let epochSeconds: Int64
    public let epochMillis: Int64
    public let isoUTC: String      // "2026-05-21T14:32:10Z"
    public let isoLocal: String    // "2026-05-21T11:32:10-03:00"
}

public enum TimestampEngine {
    public static func parse(_ raw: String) -> Result<TimestampBreakdown, ToolError>
    public static func now() -> TimestampBreakdown
}
```

- `parse`:
  1. Trim whitespace. Empty → `.invalidInput(reason: "Input is empty")`.
  2. If the trimmed string is all digits:
     - length == 10 → interpret as seconds.
     - length == 13 → interpret as milliseconds.
     - any other length → `.parseFailure(reason: "Numeric timestamp must be 10 digits (seconds) or 13 digits (milliseconds)", hint: nil)`. Do not fall through to ISO parsing.
  3. Otherwise try `ISO8601DateFormatter` with `.withInternetDateTime` (no fractional), then again with `[.withInternetDateTime, .withFractionalSeconds]`.
  4. If both ISO attempts fail → `.parseFailure(reason: "Not a valid epoch or ISO 8601 timestamp", hint: nil)`.
- `now`: return breakdown of `Date()`.
- `isoUTC` uses `ISO8601DateFormatter` with `.withInternetDateTime`. `isoLocal` uses the same formatter with `.withInternetDateTime` and a local-time-zone configuration.

**View (`TimestampTool`):** Input `TextField` + `Now` button + four `MonospaceOutput` rows (Epoch (s), Epoch (ms), ISO 8601 UTC, ISO 8601 Local). Inline `InlineErrorBanner` on failure.

**Tool conformance:**

```swift
public enum TimestampTool: Tool {
    public static let id: ToolID = "timestamp"
    public static let displayName = "Timestamp"
    public static let iconName = "clock"
    public static let keywords = ["timestamp", "epoch", "unix", "iso", "date", "time"]
    public static let category: ToolCategory = .time
}
```

**Test cases:**
- `"1716300730"` (10 digits) → epochSeconds = 1716300730, epochMillis = 1716300730000, isoUTC fixed.
- `"1716300730123"` (13 digits) → epochMillis = 1716300730123.
- `"2026-05-21T14:32:10Z"` → epoch matches expected.
- `"2026-05-21T14:32:10.500Z"` (fractional) → epochMillis with `.500` part preserved.
- `"not a date"` → `parseFailure`.
- `"171630073"` (9 digits) and `"17163007301"` (11 digits) → `parseFailure` (must be 10 or 13).
- `""` → `invalidInput`.
- `now()` returns `epochSeconds` within ±1s of `Int64(Date().timeIntervalSince1970)`.

## Dependencies

**None added.** All v0.1 dependencies (none, in fact — v0.1 ships with zero external SPM deps in the four tool packages) remain unchanged. `CryptoKit` and `Security` (for `SecRandomCopyBytes`) are Apple system frameworks already available.

## Error handling

Unchanged from v0.1. Every engine returns `Result<T, ToolError>` with one of:

- `.invalidInput(reason:)` for empty input, out-of-range counts, etc.
- `.parseFailure(reason:hint:)` for malformed inputs the engine couldn't interpret.

Views render `InlineErrorBanner` on `.failure` and clear output. Storage / unsupported / dependencyMissing branches are not used by these tools.

## Testing

Same shape as v0.1:

- **Engine unit tests** (one `XxxToolKitTests` per tool) — listed under "Test cases" in each tool section above. Run via `swift test` per package, parallelized in CI by the existing GitHub Actions workflow.
- **Smoke E2E** unchanged — the existing `Tests/DevTrayAppUITests/SmokeTests.swift` continues to assert the app launches and stays alive.
- **Manual checklist** (release gate, per Task 17 of v0.1): exercise each new tool once in the popover before tagging.

No new test infrastructure.

## Open questions

None blocking. Implementation choices the plan will make:

- Exact `iconName` SF Symbols may be tweaked when seen in the sidebar.
- ULID monotonic guarantee within a single millisecond batch: the engine can sort random suffixes ascending to preserve sort order for back-to-back generations; the test asserts shared timestamp prefix but not strict monotonicity (deferred).
- Whether the UUID tool's "Generate" runs on Picker change or only on button press — likely button press, to avoid wasted work while the user adjusts count.
