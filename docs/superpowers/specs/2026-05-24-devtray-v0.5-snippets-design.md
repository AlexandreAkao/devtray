# DevTray v0.5 — SnippetsTool (design)

**Date:** 2026-05-24
**Status:** Approved (brainstorm)
**Predecessor:** v0.4 Spotlight (`docs/superpowers/plans/2026-05-23-devtray-v0.4-spotlight.md`)
**Parent spec:** `docs/superpowers/specs/2026-05-20-devtray-design.md` (sections "Release phasing → v0.5", "Data and persistence → Schema (snippets, snippets_fts)", "UI surfaces → Tool views")

## Goal

Ship `SnippetsTool` — the first DevTray tool with heavy persistence. A personal,
local, full-text-searchable snippet library with tags, favorites, and use
tracking. No telemetry, no remote sync; the only tool inputs DevTray ever
persists are the snippets the user explicitly saves.

## Scope

**In:**
- `Snippet` model + `SnippetStore` protocol + `InMemorySnippetStore` + `SnippetStoreEnvironment` in `DevTrayCore`.
- `SQLiteSnippetStore` (GRDB) in `DevTrayStorage`, plus the `v2_snippets` migration (`snippets` table, indexes, `snippets_fts` FTS5 virtual table + sync triggers).
- `SnippetsTool` package (`SnippetsToolKit` engine + `SnippetsTool` view) under `Packages/Tools/`.
- App wiring: construct `SQLiteSnippetStore.openDefault()` in `DevTrayApp.swift`, inject via a new `\.snippetStore` environment key.
- Registration of `SnippetsTool` in the `ToolRegistry`.

**Out (deferred to v0.6):**
- Export / import snippets as JSON (and the Settings → General tab that hosts it).
- The ⌘↩ Spotlight fix (`currentSelectedID()` returns nil). Snippets ships with ⌘↩ as a known limitation, same as v0.4.
- CloudKit / cross-machine sync.

## Decisions (from brainstorm 2026-05-24)

1. **SwiftPM empty-`Sources` gotcha → baked stub.** `Package.swift` declares both targets from the start. The engine task's first action writes a 2-line placeholder `Sources/SnippetsTool/SnippetsTool.swift` so `swift test` finds source for the declared `SnippetsTool` target; the view task overwrites it. (Carries the gotcha first seen in v0.2: a declared target with an empty `Sources/<target>/` dir fails `swift test` with "Source files for target … should be located under …".)
2. **Tags → JSON inline.** `tags TEXT NOT NULL DEFAULT '[]'` on the `snippets` row, denormalized into `snippets_fts`. No junction table. Fine at personal-library scale; central to UX, not to relational integrity.
3. **FTS5 sync triggers → GRDB helper.** Create the FTS table with GRDB's `create(virtualTable:using: FTS5())` + `synchronize(withTable: "snippets")`, which generates the three external-content sync triggers (INSERT/UPDATE/DELETE) — including the easy-to-botch `'delete'` incantation. Behavior is tested regardless of how triggers are created.
4. **⌘↩ Spotlight fix → deferred to v0.6.** Keeps v0.5 focused on the persistence milestone; avoids stacking a fragile SwiftUI↔AppKit fix onto the heaviest tool release.

### Architecture deviation from the kickoff (resolved)

The kickoff text said `SnippetsToolKit` depends on `DevTrayCore + DevTrayStorage`.
We instead follow the established `UsageStore` pattern: the concrete store is
constructed in the **app target** and injected via `@Environment`, so the Kit
only ever references the **`SnippetStore` protocol**. Therefore
**`SnippetsToolKit` depends on `DevTrayCore` only**, like every other ToolKit.
This keeps the engine pure and testable against `InMemorySnippetStore`.

## Layers

### DevTrayCore (protocol + model + doubles + environment)

```swift
public struct Snippet: Identifiable, Hashable, Sendable {
    public typealias ID = String
    public let id: ID                 // UUID string; PRIMARY KEY
    public var title: String
    public var content: String
    public var language: String?
    public var tags: [String]
    public var isFavorite: Bool
    public let createdAt: Date
    public var updatedAt: Date
    public var useCount: Int
    public var lastUsedAt: Date?
}

public protocol SnippetStore: Sendable {
    func save(_ snippet: Snippet) async throws          // upsert on id
    func delete(id: Snippet.ID) async throws
    func all() async throws -> [Snippet]                // ordered by updated_at DESC
    func search(_ query: String) async throws -> [Snippet]   // FTS; empty query -> all()
    func incrementUseCount(id: Snippet.ID) async throws      // also stamps last_used_at = now
}
```

- **`InMemorySnippetStore`** — an `actor` conforming to `SnippetStore` for previews and Kit tests. `search` does case-insensitive substring matching over title/content/tags.
- **`SnippetStoreEnvironment`** — a SwiftUI `EnvironmentKey` whose `defaultValue` is `InMemorySnippetStore()`, exposed as `EnvironmentValues.snippetStore` (mirrors `UsageStoreEnvironment`). Use an explicit type annotation when reading it cross-module (`@Environment(\.snippetStore) private var snippetStore: any SnippetStore`) — v0.4 lesson on cross-module key-path inference.

### DevTrayStorage (`SQLiteSnippetStore`)

Mirrors `SQLiteUsageStore`:

```swift
public final class SQLiteSnippetStore: SnippetStore, @unchecked Sendable {
    public init(dbQueue: DatabaseQueue)
    public static func openDefault() throws -> SQLiteSnippetStore  // DatabaseOpener + Migrations
}
```

- Reuses `DatabaseOpener` (shared default URL + corruption recovery) and the single `DatabaseMigrator` populated by `Migrations.register`.
- `save` = `INSERT … ON CONFLICT(id) DO UPDATE …`; tags serialized via `SnippetTagCodec`. Sets/updates `updated_at`.
- `all` = `SELECT … ORDER BY updated_at DESC`; decodes tags JSON.
- `search` = join `snippets` with `snippets_fts` via `MATCH`, ordered by `bm25(snippets_fts)` then `updated_at DESC`. Empty/whitespace query short-circuits to `all()`. Query string sanitized by `FTSQuery` before reaching SQLite.
- `incrementUseCount` = `UPDATE snippets SET use_count = use_count + 1, last_used_at = ? WHERE id = ?`.
- All methods `throw` on failure (the protocol is throwing — unlike `UsageStore.record`, which swallows errors for fire-and-forget usage logging).
- `os.Logger(subsystem: "com.devtray.app", category: "storage")` for diagnostic logging.

### Migration — `v2_snippets`

Added as a second `registerMigration` entry in the existing `Migrations.swift`.
Does **not** touch `tool_usage`.

```sql
CREATE TABLE snippets (
    id            TEXT PRIMARY KEY,
    title         TEXT NOT NULL,
    content       TEXT NOT NULL,
    language      TEXT,
    tags          TEXT NOT NULL DEFAULT '[]',
    is_favorite   INTEGER NOT NULL DEFAULT 0,
    created_at    REAL NOT NULL,
    updated_at    REAL NOT NULL,
    use_count     INTEGER NOT NULL DEFAULT 0,
    last_used_at  REAL
);
CREATE INDEX idx_snippets_updated  ON snippets(updated_at DESC);
CREATE INDEX idx_snippets_favorite ON snippets(is_favorite, updated_at DESC);
```

Then the FTS5 table via GRDB (generates the three sync triggers and back-fills
from existing rows — a no-op here since `snippets` is empty at migration time):

```swift
try db.create(virtualTable: "snippets_fts", using: FTS5()) { t in
    t.synchronize(withTable: "snippets")
    t.column("title")
    t.column("content")
    t.column("tags")
}
```

## Data flow

1. `DevTrayApp` constructs `SQLiteSnippetStore.openDefault()` once (a `makeSnippetStore()` helper alongside `makeUsageStore()`), injects via `.environment(\.snippetStore, snippetStore)`.
2. `SnippetsToolView` reads `@Environment(\.snippetStore)`, builds a `SnippetsModel`, loads `all()` on appear.
3. Search field text → debounced → `store.search(query)` → updates the list.
4. Favorites filter toggle → filters the current list (favorites have an index for ordering, but filtering the loaded set is sufficient at this scale).
5. Editor Save/Delete → `store.save`/`store.delete` → reload list.
6. "Copy" → write content to `NSPasteboard` + `store.incrementUseCount(id:)`.
7. "New" → blank snippet in the editor; first Save persists it (UUID assigned on creation).

**Clipboard preload:** `SnippetsTool` opts out of smart-paste. It is the
"navigate without preload" tool, so opening it from Spotlight or the popover
shows the list rather than pre-filling content from the clipboard.

## UI surfaces (view target)

- **`SnippetsListView`** — search field (FTS), favorites filter toggle, "New" button, and rows showing title, language badge, favorite star, and tags. Reuses DevTrayUI styling.
- **`SnippetEditorView`** — title field, content via the existing `CodeEditor` (Highlightr) with a language picker, a tags field, a favorite toggle, and Save / Delete / Copy actions.
- **Errors** surface through the existing `InlineErrorBanner` from DevTrayUI.
- Keyboard handling follows v0.4 lessons: keys that must work while a `TextField` is focused (esc, ↑/↓) attach to the field or the responder chain, not a detached `.onKeyPress`.

## Error handling

- Store methods throw; the view model catches and shows `InlineErrorBanner`.
- Database corruption at open is handled by `DatabaseOpener` (rename to `*.corrupted-<date>`, recreate) — already in place.
- `FTSQuery` sanitizes arbitrary user input so a stray quote or operator can't produce an FTS5 `MATCH` syntax error; empty input bypasses FTS entirely.

## Testing

**DevTrayStorage (`SQLiteSnippetStore` vs in-memory `DatabaseQueue`):**
- CRUD round-trip: save → all → fields match (including tags JSON and `language` nil).
- Upsert: saving the same id twice updates rather than duplicates.
- FTS reflects mutations (validates the synchronize triggers): insert → searchable; update title/content → search reflects new text; delete → no longer found.
- `search` ordering and empty-query → `all()` behavior.
- `incrementUseCount` increments and stamps `last_used_at`.
- **Migration fixture v1→v2:** open a v1 database, seed `tool_usage` rows, run the migrator to v2, then assert: (a) the `tool_usage` rows are intact (no corruption), (b) a snippet inserted post-migration round-trips through `all()` and is found via `search`.

**SnippetsToolKit:**
- `Tool` conformance metadata (id `snippets`, category, title, icon).
- `SnippetTagCodec` round-trip ([String] → JSON → [String]), including empty and unicode.
- `FTSQuery` sanitization (quotes/operators neutralized; empty input flagged).
- `SnippetsCoordinator` CRUD + search against `InMemorySnippetStore`.

**View target:** light — the project's pattern tests Kit + storage, not heavy UI.

## Lessons carried from v0.4

- A method observable via a generic constraint (`<T: Tool>`) must be declared in the protocol body, not only an extension (extension-only = static dispatch = overrides ignored).
- `@Environment(\.snippetStore)` cross-module reads use an explicit `: any SnippetStore` type annotation.
- After cross-module SwiftPM work, Clean Build Folder (`⌥⇧⌘K`) before the first manual run; trust `swift test` + `xcodebuild` over transient SourceKit warnings.
