## Goal

Three deliverables, in priority order:

1. **Snippet export/import (JSON) + Settings → General tab.** The primary piece. Per the design spec (`2026-05-20-devtray-design.md`, "Backup / migration between machines"): Settings → **Export snippets** writes JSON; **Import snippets** reads it back. This is the v1 path for moving a snippet library between Macs before CloudKit.
2. **`⌘↩` Spotlight fix.** Carried since v0.4. With 8 tools (Snippets being the "navigate without preload" tool), it now earns its place.
3. **SnippetEditorView polish left over from v0.5:** favorite toggle in the editor; `InlineErrorBanner` in Snippets.

## Brainstorm decisions

| Question | Decision | Why |
|---|---|---|
| Export JSON shape | **Versioned envelope** `{ "version": 1, "exportedAt": …, "snippets": [...] }` | We ship public via Sparkle; format stability is an open question in the design spec. The wrapper costs ~3 lines and gives the import a version to dispatch on for future format migrations. |
| Import merge strategy | **Merge by id (upsert)** | Reuses the existing `save()` upsert (same ids overwrite, new ids add). Non-destructive, no new protocol method, no destructive-confirmation dialog. `created_at` preserved by the existing upsert. |
| Export fidelity | **Full fidelity** | Use case is machine-to-machine migration. Synthesized `Codable` carries every field, so usage stats (`useCount`/`lastUsedAt`) and "recently used" survive the move. |
| `⌘↩` scope | **Include in v0.6** | Debt since v0.4; matters more with 8 tools. Investigation (below) turned the chosen approach into a requirement, not just a preference. |

## Packaging (architecture decision, not a brainstorm question)

Export/import is **not** a Tool package. The codec (`SnippetArchive`) lives in `DevTrayCore` alongside `Snippet`/`SnippetStore`, pure and unit-testable. The file pickers (`NSSavePanel`/`NSOpenPanel`) and the `GeneralTab` live in the app target (`App/DevTrayApp/`), following the `ShortcutsTab`/`AboutTab` pattern in `SettingsView`. Import does **not** need `deleteAll()` — merge-by-id reuses `save()`.

## Components

### 1. `SnippetArchive` codec (DevTrayCore, pure & testable)

`Snippet` gains `Codable` conformance (synthesized — full fidelity, all stored fields).

```swift
public enum SnippetArchive {
    public static let currentVersion = 1
    struct Envelope: Codable { let version: Int; let exportedAt: Date; let snippets: [Snippet] }

    public static func encode(_ snippets: [Snippet], exportedAt: Date) throws -> Data
    public static func decode(_ data: Data) throws -> [Snippet]
}
```

- **Encoder:** `.iso8601` date strategy, `.prettyPrinted` + `.sortedKeys` (stable, diffable files).
- **Decoder:** branches on `version`. `1` → return `snippets`. Unknown version → `throw ToolError.parseFailure(reason: "Unsupported export version \(v)", hint: "This file was made by a newer DevTray.")`. This branch is the migration seam the envelope buys.
- **Malformed JSON:** caught and rethrown as `ToolError.parseFailure`.
- No store dependency — the codec is sync and pure.

Import orchestration stays in the app: `decode → for each: try await store.save($0)`. Merge-by-id falls out of the existing `save()` upsert for free; `created_at` is preserved.

### 2. `GeneralTab` + file pickers (app target)

New `GeneralTab` inside `DevTrayApp.swift`, registered in `SettingsView`'s `TabView` between Shortcuts and About: `Label("General", systemImage: "gearshape")`.

- **Export snippets…** → `NSSavePanel` (default name `DevTray-snippets-<date>.json`) → `store.all()` → `SnippetArchive.encode` → write `Data`.
- **Import snippets…** → `NSOpenPanel` (json only) → read → `SnippetArchive.decode` → loop `save()` → report "Imported N snippets" or show the error inline.

Reads `@Environment(\.snippetStore) var store: any SnippetStore` (explicit type annotation — cross-module inference rule from v0.5). No entitlement changes: the app is unsandboxed (only `com.apple.security.network.client` is set).

### 3. Editor polish (SnippetsTool)

- **Favorite toggle in editor:** add `@State isFavorite`, a star toggle in the editor's button row, set in `loadFields()`, persisted through `current()` on Save — consistent with how title/content/language/tags already flow through Save.
- **InlineErrorBanner:** change `SnippetsModel.errorMessage: String?` → `error: ToolError?`. The `run` catch maps `(error as? ToolError) ?? .storageFailure(message: error.localizedDescription)`. `SnippetsListView` swaps the red `Text` (lines 29–34) for `InlineErrorBanner(error:)`. `ToolError.storageFailure(message:)` already exists in `DevTrayCore` — no new case needed.

### 4. `⌘↩` Spotlight fix

**Why the old plumbing was a no-op:** `SpotlightController.currentSelectedID()` hardcodes `return nil`, but the deeper issue is that the AppKit path could only fire `preloadBus.send`, never flip the SwiftUI inline `mode` (`SpotlightSearchView.mode`). `⌘↩` means "open the selected tool inline **without** preloading clipboard," and the mode switch lives in SwiftUI. So the fix must be handled SwiftUI-side.

- **Plan A (primary — removes the fragile bridge):**
  - `SpotlightPanel.keyDown`: stop intercepting `⌘↩`; let it fall through to `super.keyDown` like bare `↩` already does. Delete the `onReturnWithCommand` property.
  - `SpotlightController`: delete `handleSubmitWithCommand()`, `currentSelectedID()`, and the `makePanel` `onReturnWithCommand` wiring.
  - `SpotlightSearchView`: handle `⌘↩` via `onKeyPress` inspecting `press.modifiers.contains(.command)` → `handleReturn(withPreload: false)`. `handleReturn` already fires `onSubmit` and sets `mode = .tool(id)`.
  - Rationale: nothing in DevTray binds `⌘↩` as a menu key equivalent (which is why the panel's `keyDown` already receives it today), so once the panel stops swallowing it the event should reach SwiftUI's `onKeyPress`.
- **Plan B (fallback — only if manual smoke shows `⌘↩` never reaches SwiftUI):** keep the panel interception; the controller holds the shared `SpotlightViewModel` reference (it already constructs the vm in the content closure) and signals a trigger the view observes to call `handleReturn(withPreload: false)`. Documented; not built unless A fails the smoke test.

## Carried v0.5 lessons relevant here

- XcodeGen is the source of truth (`DevTray.xcodeproj` is gitignored, generated from `project.yml`). New app-target files or test targets require `xcodegen generate`.
- Version bump lives in two places (`project.yml` + `Info.plist`).
- `XCTAssert(try await …)` does not compile — break async into a `let` before the assert.
- `@Environment(\.xxx)` cross-module always needs an explicit type annotation (`: any SnippetStore`).
- The Spotlight `NSHostingView` injects only `\.snippetStore` (added v0.5), not `\.usageStore`/`\.preloadBus`. If any new env must reach hosted tool views, inject in both `PopoverRoot` and `SpotlightController.installContent`. (v0.6 adds no new env to the hosted view — the `⌘↩` fix is SwiftUI-internal.)
- `gh` dual account: repo is `AlexandreAkao` (SSH push works); active `gh` is usually `AlexandreAkiraAkao` — `gh auth switch --user AlexandreAkao` before any API op, restore after.

## Out of scope

- CloudKit sync (later).
- `deleteAll()` / replace-all import (merge-by-id chosen).
- Content-only / "share a single snippet" export (full-fidelity library export only).
- App sandboxing (would require user-selected-file entitlements; not needed while unsandboxed).
