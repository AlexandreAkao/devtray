# DevTray v0.4 — Design Document

**Date:** 2026-05-23
**Status:** Design (awaiting plan)
**Author:** Brainstorming session with user
**Predecessor:** [v0.3 storage](2026-05-21-devtray-v0.3-storage-design.md) (shipped as `v0.3.0` on 2026-05-21)

## Summary

v0.4 adds the **Spotlight panel + global hotkey + smart paste** — the second invocation surface beside the menubar popover. A configurable global hotkey (default `⌥⌘Space`) opens a borderless floating panel with a search field and a result list. Each tool declares a static `clipboardMatch(_:)` matcher so the panel can promote the right tool when the clipboard contains a recognized format (JWT, JSON, URL, Base64, hash, UUID, timestamp). Ranking is modal by context: with no query, smart-paste wins and the rest of the list comes from `topTools`; with a query, fuzzy match drives order and smart-paste degrades to a "from clipboard" badge on the matching row. Pressing `↩` closes the panel and opens the popover with the selected tool, pre-filling its input via a lightweight `PreloadBus`. No persistence schema changes — v0.4 reads from the same `UsageStore` shipped in v0.3.

## Goals

- Ship the second invocation surface (Spotlight panel) so DevTray is reachable without clicking the menubar icon.
- Detect and promote clipboard contents with per-tool matchers, keeping format knowledge co-located with each tool.
- Keep the ranker pure and testable in `DevTrayCore` (no AppKit, no view code).
- Reuse v0.3 infrastructure (`UsageStore`, `ToolRegistry`, popover, `FrequentToolChip`) instead of duplicating tool hosting.
- Add the Settings scene with a Shortcuts tab so users can rebind the hotkey from a familiar location.

## Non-goals

- Hosting tool views inside the Spotlight panel (Spotlight stays a chooser; the popover stays the host). Inline tool view in Spotlight is deferred to v0.5+ if ever.
- A General preferences tab (launch at login, popover behavior). Deferred to v0.5+.
- A Tools enable/disable tab. Deferred to v0.6 when the catalog grows.
- New tools. v0.4 ships against the 7 tools from v0.2.
- Schema changes. v0.4 only reads `tool_usage`.
- Inline syntax-aware previews of clipboard content. The pill says "from clipboard"; the actual preview happens after `↩`.

## Decisions (from brainstorming)

| # | Decision | Choice | Rationale |
|---|---|---|---|
| 1 | Global hotkey library | `KeyboardShortcuts` (Sindre Sorhus) | Ships a SwiftUI `Recorder` view; Carbon-based registration; no Accessibility permission |
| 2 | Smart-paste detection location | Per-tool via `Tool` protocol extension | Format knowledge lives with each tool; `DevTrayCore` stays domain-agnostic |
| 3 | Confidence model | Two-level enum (`weak` / `strong`) | YAGNI vs. `Double` score; covers every planned tool unambiguously |
| 4 | Spotlight ranking | Modal by context (no query → smart-paste + topTools; with query → fuzzy + frequency, smart-paste degraded to pill) | Typing expresses explicit intent; not typing infers from context |
| 5 | Smart-paste UX | Top result with "from clipboard" pill | Discreet, lets user see alternatives, consistent with macOS Spotlight |
| 6 | Spotlight role | Chooser that opens the popover (`Approach 1`) | Smallest blast radius; reuses popover infra; defers tool-host duplication |
| 7 | Packaging | No new package; code distributed across `DevTrayCore`, `DevTrayUI`, `DevTrayApp`, `Tools/*` | Surface too small to justify a 5th package |
| 8 | Preferences scope | Shortcuts + About tabs only | Minimum needed to expose `KeyboardShortcuts.Recorder`; other tabs deferred |
| 9 | Plan B for popover open | Accept private-ish `NSStatusItem.button?.performClick(nil)`; document fallback to in-panel tool view if it fails | Refactoring `MenuBarExtra` away in v0.4 is out of scope |

## Smart-paste — `Tool` protocol extension

### New types (in `DevTrayCore`)

```swift
public struct ClipboardMatchScore: Sendable, Equatable {
    public enum Confidence: Int, Sendable, Comparable {
        case weak
        case strong
        public static func < (l: Self, r: Self) -> Bool { l.rawValue < r.rawValue }
    }
    public let confidence: Confidence
    public init(_ confidence: Confidence) { self.confidence = confidence }
}

public extension Tool {
    static func clipboardMatch(_ clipboard: String) -> ClipboardMatchScore? { nil }
}
```

Default implementation returns `nil` so tools that don't implement the matcher silently opt out. Each Tool Kit overrides:

```swift
extension JWTTool {
    public static func clipboardMatch(_ clipboard: String) -> ClipboardMatchScore? {
        let trimmed = clipboard.trimmingCharacters(in: .whitespacesAndNewlines)
        guard trimmed.hasPrefix("eyJ"),
              trimmed.split(separator: ".").count == 3 else { return nil }
        return ClipboardMatchScore(.strong)
    }
}
```

### Per-tool matchers

| Tool | Pattern | Confidence |
|---|---|---|
| JWT | starts with `eyJ`, exactly 3 dot-separated segments of base64url chars | strong |
| JSON | trimmed starts with `{` or `[` and ends with matching `}` or `]` (no full parse — cheap heuristic) | strong |
| Base64 | length ≥ 8, length % 4 == 0, base64 alphabet, NOT a JWT (no dots) | weak |
| URL | starts with a recognized scheme (`http://`, `https://`, `ftp://`, etc.) or matches `scheme://host` shape | strong |
| Hash | hex-only string of length 32 / 40 / 64 / 128 | weak |
| UUID | matches UUID regex `^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$` (case-insensitive) or ULID `^[0-9A-HJKMNP-TV-Z]{26}$` | strong |
| Timestamp | 10 or 13 digit integer in 2001–2100 unix range | weak |

Matchers are pure functions, testable inside the existing `XxxToolKitTests` package without involving Spotlight code.

## Spotlight ranker

### Result type

```swift
public struct SpotlightResult: Sendable, Equatable {
    public let toolID: ToolID
    public let fromClipboard: Bool   // drives the pill in UI
    public init(toolID: ToolID, fromClipboard: Bool) { … }
}
```

### Algorithm

The ranker reads tools from `ToolRegistry.tools: [AnyTool]` (the existing published array). It needs a way to call each tool's static `clipboardMatch` from an `AnyTool` instance — see "Calling `clipboardMatch` through `AnyTool`" below.

```swift
@MainActor
public final class SpotlightRanker {
    private let registry: ToolRegistry
    private let usage: any UsageStore

    public func rank(query: String, clipboard: String?, limit: Int = 8) async -> [SpotlightResult] {
        let q = query.trimmingCharacters(in: .whitespacesAndNewlines)

        if q.isEmpty {
            // No query: smart-paste promotes; rest comes from topTools.
            let matchers = registry.tools.compactMap { tool -> (ToolID, ClipboardMatchScore)? in
                guard let clipboard, let score = tool.clipboardMatch(clipboard) else { return nil }
                return (tool.id, score)
            }
            if let top = pickTopMatcher(matchers, usage: usage) {
                let rest = (try? await usage.topTools(window: .lastDays(7), limit: limit - 1, now: .now))
                    ?? []
                let merged = [SpotlightResult(toolID: top, fromClipboard: true)]
                    + rest.filter { $0.toolID != top }
                          .map { SpotlightResult(toolID: $0.toolID, fromClipboard: false) }
                return Array(merged.prefix(limit))
            }
            // No clipboard match: pure topTools.
            let ranks = (try? await usage.topTools(window: .lastDays(7), limit: limit, now: .now))
                ?? []
            return ranks.map { SpotlightResult(toolID: $0.toolID, fromClipboard: false) }
        }

        // Query: fuzzy match drives, smart-paste is just a badge.
        let matched = registry.tools.compactMap { tool -> (ToolID, Int)? in
            guard let score = fuzzyScore(query: q, displayName: tool.displayName, keywords: tool.keywords)
                else { return nil }
            return (tool.id, score)
        }
        let usageRank = (try? await usage.topTools(window: .allTime, limit: 100, now: .now))
            .map { Dictionary(uniqueKeysWithValues: $0.enumerated().map { ($1.toolID, $0) }) }
            ?? [:]
        let ordered = matched.sorted { a, b in
            if a.1 != b.1 { return a.1 > b.1 }                                // fuzzy DESC
            let aRank = usageRank[a.0] ?? .max
            let bRank = usageRank[b.0] ?? .max
            if aRank != bRank { return aRank < bRank }                        // usage rank ASC
            return a.0.rawValue < b.0.rawValue                                // tie-break by id
        }
        return ordered.prefix(limit).map { toolID, _ in
            let badge: Bool
            if let clipboard, let tool = registry.find(byID: toolID) {
                badge = tool.clipboardMatch(clipboard) != nil
            } else {
                badge = false
            }
            return SpotlightResult(toolID: toolID, fromClipboard: badge)
        }
    }
}
```

> `pickTopMatcher` is a private helper that sorts matchers by `confidence` DESC, then by `topTools(.allTime)` rank ASC, then by `tool_id` ASC; returns the first `ToolID`. Pseudo-code above is illustrative.

### Calling `clipboardMatch` through `AnyTool`

`AnyTool` is the value-type erasure used everywhere outside Tool packages. Today it captures `id`, `displayName`, `iconName`, `keywords`, `category`, and a `@MainActor` `makeView` closure. v0.4 adds one captured closure:

```swift
public struct AnyTool: Identifiable, Hashable, Sendable {
    // … existing fields …
    private let _clipboardMatch: @Sendable (String) -> ClipboardMatchScore?

    public init<T: Tool>(_ tool: T.Type) {
        // … existing assignments …
        self._clipboardMatch = { T.clipboardMatch($0) }
    }

    public func clipboardMatch(_ clipboard: String) -> ClipboardMatchScore? {
        _clipboardMatch(clipboard)
    }
}
```

Tools that don't override the default protocol method get `nil` automatically. No registry changes needed.

### Fuzzy match

```swift
/// Returns nil if not a match. Higher score = better.
/// - Tries displayName first (case-insensitive subsequence). +10 if query is a prefix.
/// - If no displayName match, tries each keyword. Match on a keyword scores lower than displayName.
/// - Returns highest of the candidate scores, or nil.
internal func fuzzyScore(query: String, displayName: String, keywords: [String]) -> Int?
```

~40 lines, no external dependency. Unit-tested with positive, negative, prefix-bonus, keyword-only, and tie-break cases.

### Tie-break determinism

Ties on every signal collapse to `tool_id ASC`, matching the `topTools` tie-break shipped in v0.3.

### Lifecycle

| Trigger | Behavior |
|---|---|
| `⌥⌘Space` global | `SpotlightController.toggle()`. If visible, close. Else read clipboard, position, show, focus search field. |
| `esc` | Close, no side effects. |
| `↩` on selected row | `bus.send(PreloadPayload(toolID, text: clipboard ?? ""))`; close; trigger popover open. |
| `⌘↩` on selected row | `bus.send(PreloadPayload(toolID, text: nil))`; close; trigger popover open. (Navigation without preload.) |
| `↑` / `↓` | Move selection. List wraps at edges. |
| `⌘N` / `⌘[1–9]` | Out of scope for v0.4. |
| Click outside / window resigns key | Close. |

### Positioning

Centered horizontally on the screen with the cursor, top edge at 25 % of screen height (macOS Spotlight convention). Recomputed every open from `NSScreen` containing the cursor — no persisted position.

### `SpotlightRow`

Takes a resolved `AnyTool` (looked up by the view model from `ToolRegistry.find(byID:)` for each result) plus the `SpotlightResult`:

```
HStack(spacing: 10) {
    Image(systemName: tool.iconName)
        .frame(width: 22, height: 22)
        .foregroundStyle(.secondary)
    Text(tool.displayName).font(.body)
    Spacer()
    if result.fromClipboard {
        FromClipboardPill()                   // 10pt uppercase "from clipboard", accent color
    }
}
.padding(.horizontal, 12).padding(.vertical, 8)
.background(isSelected ? Color.accentColor.opacity(0.18) : .clear)
.cornerRadius(8)
```

Single-line layout (no secondary description). The earlier mockup showed a meta line — dropped for v0.4 because `Tool` doesn't carry a description field and adding one is out of scope. Adding `shortDescription` to the `Tool` protocol can ship later if rows feel too sparse in practice.

### View model

`SpotlightViewModel` (`@MainActor`, `ObservableObject` to match the v0.3 pattern) holds:
- `query: String` (debounced 80 ms)
- `clipboard: String?` (captured once at panel open)
- `rows: [(SpotlightResult, AnyTool)]` — published. The view model resolves each `SpotlightResult.toolID` to an `AnyTool` via `ToolRegistry.find(byID:)` before publishing, so rows render without reaching back into the registry.
- `selectedID: ToolID?` (defaults to the first row on each re-rank; preserved across re-ranks if still present)

On every change of `query` (post-debounce), it calls `ranker.rank(query:clipboard:limit:)`, resolves each result to `AnyTool`, and assigns `rows`. Results missing from the registry (shouldn't happen — guard against it) are dropped silently.

### Wiring

```swift
@main
struct DevTrayApp: App {
    @StateObject private var registry = makeRegistry()
    private let usageStore: any UsageStore = makeUsageStore()
    private let preloadBus = PreloadBus()
    @State private var spotlight: SpotlightController?

    init() {
        let controller = SpotlightController(/* registry, usage, preloadBus */)
        _spotlight = State(initialValue: controller)

    }

    var body: some Scene {
        MenuBarExtra("DevTray", systemImage: "wrench.adjustable") {
            PopoverRoot()
                .environmentObject(registry)
                .environment(\.usageStore, usageStore)
                .environment(\.preloadBus, preloadBus)
        }
        .menuBarExtraStyle(.window)

        Settings { SettingsView() }   // existing scene, now with ShortcutsTab added inside
    }

```

The actual `SpotlightController` init signature and the precise way the controller is captured by the `KeyboardShortcuts` closure are implementation details — the plan will refine them once the controller exists.

### Popover-open mechanism (with Plan B)

`SpotlightController` triggers the popover via `NSStatusItem.button?.performClick(nil)`. If `MenuBarExtra`'s opaque status item makes that fail at runtime, fall back to **Plan B**: render the selected tool inside the Spotlight panel itself by adding a second internal view state (`.list` / `.tool`) to `SpotlightSearchView`. The list view keeps using `SpotlightRow`; the tool view embeds the tool's existing SwiftUI view (the same one `PopoverRoot` would show). Plan B is documented in the plan as a mid-flight fix, not pre-built.

### Entitlements

No new entitlements. `KeyboardShortcuts` uses Carbon `RegisterEventHotKey`, which works under Hardened Runtime without Accessibility permission. `com.apple.security.network.client` already exists from v0.3 for Sparkle.

## `PreloadBus` — passing clipboard preload to the popover

### Types (in `DevTrayCore`)

```swift
public struct PreloadPayload: Sendable, Equatable {
    public let toolID: ToolID
    public let text: String?    // nil = navigate only, do not pre-fill
    public init(toolID: ToolID, text: String?) { … }
}

@MainActor
public final class PreloadBus: ObservableObject {
    @Published public private(set) var pending: PreloadPayload?
    public func send(_ p: PreloadPayload) { pending = p }
    public func consume() -> PreloadPayload? {
        let p = pending; pending = nil; return p
    }
    public init() {}
}

public struct PreloadBusEnvironment: EnvironmentKey {
    public static let defaultValue: PreloadBus = .init()
}
public extension EnvironmentValues {
    var preloadBus: PreloadBus {
        get { self[PreloadBusEnvironment.self] }
        set { self[PreloadBusEnvironment.self] = newValue }
    }
}
```

### Usage in tool views

```swift
struct JWTToolView: View {
    @Environment(\.preloadBus) private var preloadBus
    @State private var input = ""

    var body: some View {
        CodeEditor(text: $input, language: .jwt)
            .task(id: ObjectIdentifier(preloadBus)) {
                // Initial pass; viewModel decides whether to consume.
                applyPendingPreloadIfMatches()
            }
            .onReceive(preloadBus.$pending) { _ in
                applyPendingPreloadIfMatches()
            }
    }

    private func applyPendingPreloadIfMatches() {
        guard let payload = preloadBus.pending,
              payload.toolID == JWTTool.id,
              let text = payload.text else { return }
        input = text
        _ = preloadBus.consume()
    }
}
```

### PopoverRoot updates

`PopoverRoot` reads `\.preloadBus` and reacts to `pending`:

```swift
.onReceive(preloadBus.$pending) { payload in
    guard let payload else { return }
    selectedToolID = payload.toolID
    // text consumption happens inside the tool view, not here
}
```

This is the only change required to `PopoverRoot` from v0.3.

### Manual verification checklist (PR description)

- [ ] Press `⌥⌘Space` from another app — panel opens centered.
- [ ] Esc closes the panel.
- [ ] Type a few characters — list filters by fuzzy match.
- [ ] Copy a JWT, open panel — JWT tool is first row with "from clipboard" pill.
- [ ] Copy a URL, open panel — URL tool is first row with pill.
- [ ] Press `↩` on the JWT row — popover opens, JWT view shows token pre-filled.
- [ ] Press `⌘↩` on a row — popover opens that tool without pre-filling input.
- [ ] Reopen popover from menubar icon — input is empty (preload consumed once).
- [ ] Open Settings (`⌘,` from popover) — Shortcuts tab shows recorder; About tab shows version.
- [ ] Rebind shortcut to `⌃⌥Space` via Recorder — new shortcut works; old does not.

### Tests intentionally NOT automated

- `KeyboardShortcuts` global hotkey runtime registration (library has no test seam; covered manually).
- `SpotlightPanel` positioning and dismiss-on-resign-key (AppKit runtime).
- `NSStatusItem.button?.performClick(nil)` flow (depends on `MenuBarExtra` internals).
- Cross-process clipboard sequence (timing-sensitive).

### Test-writing rules carried from v0.3

- Window-filter ranker test and fallback test cannot share `limit` — fallback triggers whenever primary count < limit. Use `limit: 1` (or otherwise saturate) when isolating window behavior.
- Do not assert anything about `Bundle.main` from `swift test`: the xctest host resolves to Xcode itself, not the app bundle. If preload-bus default-value coverage needs `Bundle` info, inject explicitly.
- `record()` on `UsageStore` remains async non-throwing — no error surfacing from v0.4 either.
- Test naming: `func test_<scope>_<scenario>_<expectation>()` (mirroring v0.3 style).

## Risks and mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| `NSStatusItem.button?.performClick(nil)` does not open the `MenuBarExtra` popover programmatically on macOS 14 | Medium | Plan B: render the selected tool inline inside the Spotlight panel as a second internal view state. Documented in the plan, not pre-built. |
| Fuzzy match scoring produces surprising orderings on short queries | Low | 6+ ranker tests cover representative cases; algorithm is < 40 LOC and trivially auditable. |
| `KeyboardShortcuts` defaults clash with a system shortcut on user's machine | Low | Recorder shows conflicts; user can rebind. Default `⌥⌘Space` matches the v0.1 spec which already considered Spotlight collision (Spotlight is `⌘Space`). |
| Tool view `.onReceive` fires before view is mounted, losing preload | Low | `PreloadBus.pending` is sticky until `consume()`. Reopening Spotlight overwrites with the latest payload, which is the correct behavior. |
| SwiftPM target-without-sources gotcha (carried open from v0.2) | None for v0.4 | No new Tool package introduced in v0.4. Remains deferred to v0.5 when Snippets reintroduces the engine + view split. |
| `KeyboardShortcuts` SPM dep increases launch time noticeably | Very low | Library is small; loaded only by `DevTrayApp`. Measure during manual checklist if anything feels off. |

## Exit criteria

- Global hotkey opens and closes the Spotlight panel from any active app.
- Panel ranks results per algorithm: smart-paste wins with empty query; fuzzy + frequency win with query.
- Pressing `↩` opens the popover with the selected tool's input preloaded from clipboard.
- Pressing `⌘↩` opens the popover navigating to the tool without preloading input.
- Settings → Shortcuts shows the `Recorder` and rebinding works.
- All ~155 tests pass via `scripts/test.sh`.
- Manual checklist passes.
- Version bumped to `0.4.0` (`CFBundleShortVersionString`, `CFBundleVersion: "4"`).
- `Packages/DevTrayCore` exports only the new public types intended for downstream use (no leak of internal helpers).
