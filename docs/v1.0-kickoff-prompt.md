## Mudanças no backend (NÃO é brainstorm — segue do que já existe):

- **`backend/license/src/routes/webhook.ts`** — ADAPTAR pra Paddle event schema:
  - Header `X-Signature` (LS hex HMAC) → `Paddle-Signature` (formato `ts=<unix>;h1=<hex>`, signed input é `<ts>:<raw_body>`)
  - Event identifier: `meta.event_name` (LS) → `event_type` (Paddle). Same idempotency derivation works: `${event_type}:${data.id}`

  - Test mode flag: `meta.test_mode` (LS) → check `data.is_sandbox` ou similar (verify exact field via Paddle docs/sandbox events)
  - Refund event name: `order_refunded` (LS) → `transaction.refunded` ou similar (Paddle event taxonomy)

## Schema do Paddle webhook (referência — confirmar versão atual via docs):

Paddle Billing API v1 (current Paddle), event POST format:

```json
{
  "event_id": "evt_01h...",
  "event_type": "transaction.completed",
  "occurred_at": "2026-05-31T19:00:00Z",
  "notification_id": "ntf_01h...",
  "data": {
    "id": "txn_01h...",
    "status": "completed",
    "customer": { "id": "ctm_01h...", "email": "buyer@example.com" },
    "items": [{ "price": { "id": "pri_01h...", "product_id": "pro_01h..." } }],
    "details": { "totals": { "total": "1900", "currency_code": "USD" } },
    "is_sandbox": false
  }
}
```

Header `Paddle-Signature: ts=1717000000;h1=abc123def456...`

HMAC: `expected = HMAC_SHA256(secret, "${ts}:${raw_body}")`, compare with `h1` value.

## Lições que valem carregar (do v0.11 smoke, REAL POST-MORTEM):

- **Contracts entre Swift e TS NÃO foram testadas em integration** — todos 4 bugs catados em smoke seriam pegos por end-to-end test com miniflare. v1.0.1 backlog: e2e test infra com swift-script + miniflare. *(Esse problema vai REPETIR no Paddle migration se a gente só faz unit test isolated.)*
- **JWT/UUID case sensitivity** — Swift `UUID.uuidString` é UPPERCASE, JavaScript `crypto.randomUUID()` é lowercase. Backend KV é case-sensitive. **Padronizar lowercase em todos os caminhos** — Swift `.lowercased()` ANTES de qualquer URL query param OR backend faz `normalizeKey(uuid).toLowerCase()` no lookup. v0.11 escolheu backend normalize (`4070bf3`); manter pra Paddle também.
- **macOS Keychain ACL bind cdhash** — `SecItemAdd` binds item to writing binary cdhash. Unsigned/ad-hoc rebuilds = cdhash novo = previous items unreadable. v0.11 pivotou pra **file storage em `~/Library/Application Support/DevTray/`** (`5652f0e`). Strategy carryforward: any new store/secret persistence usa file storage, NÃO Keychain.
- **Webhook event_id field name é provider-specific** — LS usa `meta.webhook_id` (per-delivery, NOT stable across retries) NÃO `meta.event_id`. Paddle usa `event_id` no top-level. Generic idempotency code que assume "event_id" field falha em LS. v0.11 pegou esse bug em prod (`c46a593`). **Pra Paddle adapter**: confirmar via Paddle webhook samples que `event_id` é stable across retries (Paddle docs say yes — verify).
- **CI usa Xcode 16.2** (não 15.4 — v0.8 lesson). `sudo xcode-select -s /Applications/Xcode_16.2.app`. Brew xcodegen atual emite project format 77.
- **Scheme é `DevTrayApp`, NÃO `DevTray`** (`xcodebuild -project DevTray.xcodeproj -scheme DevTrayApp …`). `-scheme DevTray` falha com cascade enganoso "unable to resolve module dependency: DevTrayCore".
- **Bump de versão em DOIS lugares** (`project.yml` info.properties + `App/DevTrayApp/Info.plist`). v1.0 → "1.0.0" / build "12".
- **`xcodegen generate` depois de editar `project.yml`**; `.xcodeproj` é gitignored. **NÃO editar `project.pbxproj`** — arquivos novos em packages e em `App/DevTrayApp/` são auto-incluídos.
- **`gh` dual-account:** repo é AlexandreAkao (SSH push sempre); `gh` costuma default AlexandreAkiraAkao — `gh auth switch --user AlexandreAkao` antes de PR/secrets/release ops, restaurar depois.
- **Padrão de release (v0.8/v0.9/v0.10/v0.11):** PR merge no main + tag anotada manual (`vX.Y.Z`) dispara `release.yml`. **`gh pr merge` usa `--rebase` (NÃO `--merge`)** pra ff-equivalente.
- **Local-vs-origin SHA divergence depois de `gh pr merge --rebase`** é normal se você commitou local em main sem push (spec+plano feitos in-place). Recovery: `git pull --rebase` deduplica via patch-id.
- **`SPARKLE_PRIVATE_KEY` + `SUPublicEDKey` continuam intactos.** **DO NOT recriar.** License keypair também NÃO recria — está safe em CF Workers secret + 1Password backup.
- **App é unsandboxed.** Entitlements file tem `com.apple.security.network.client` mas é benign quando `com.apple.security.app-sandbox` ausente.
- **Code-signing já está real (v0.10).** Team ID `6745SC233Q`. Próximos releases auto-update Sparkle limpos a partir do v0.10 → v0.11 → v1.0 → … (Sparkle 2 signing consistency check vai aceitar; mesmo Team ID + mesmo bundle ID `com.devtray.app`).
